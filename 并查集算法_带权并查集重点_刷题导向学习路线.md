# 并查集到底解决什么问题？

并查集主要解决动态连通性问题。

它支持两个核心操作：

```text
find(x)：查询 x 所在集合的代表元素。
merge(x, y)：把 x 和 y 所在集合合并。
```

最典型的问题是：

```text
给定若干元素，反复合并两个元素所在集合，并询问两个元素是否属于同一集合。
```

普通并查集只能维护：

```text
两个元素是否连通 / 是否属于同一集合。
```

带权并查集进一步维护：

```text
同一集合中两个元素之间的相对关系。
```

例如：

- 两点之间距离；
- 两点之间奇偶关系；
- 两点之间模 3 关系；
- 两个前缀和之间的差值；
- 两个变量之间的差值约束；
- 集合内元素排列位置关系。

# 普通并查集、扩展域并查集、带权并查集的区别

## 普通并查集

只维护：

```text
x 和 y 是否在同一集合。
```

适合：

```text
连通块数量
亲戚关系
道路连通
等式约束
最小生成树 Kruskal
```

## 扩展域并查集

把一个元素拆成多个状态。

例如二元关系：

```text
x
x 的反面
```

如果 `x` 和 `y` 是敌人，就可以合并：

```text
x 与 y 的反面
x 的反面 与 y
```

适合：

```text
敌友关系
二分关系
种类关系
食物链的 3 倍拆点写法
```

## 带权并查集

在并查集的边上维护权值。

通常定义：

```text
fa[x]：x 的父节点
d[x]：x 到 fa[x] 的关系值
```

路径压缩后，`d[x]` 会被更新为：

```text
x 到根节点的关系值
```

适合：

```text
距离
奇偶
模 3 关系
前缀和差值
变量差分约束
队列相对位置
```

## 普通并查集模板

```cpp
const int N = 200000 + 10;

int fa[N], sz[N];

void init(int n) {
    for (int i = 1; i <= n; i++) {
        fa[i] = i;
        sz[i] = 1;
    }
}

int find(int x) {
    if (fa[x] == x) return x;
    return fa[x] = find(fa[x]);
}

bool merge(int x, int y) {
    int rx = find(x);
    int ry = find(y);

    if (rx == ry) return false;

    if (sz[rx] < sz[ry]) swap(rx, ry);

    fa[ry] = rx;
    sz[rx] += sz[ry];

    return true;
}
```

## 连通块数量维护

如果初始有 `n` 个点，连通块数量为：

```text
cnt = n
```

每次成功合并两个不同集合：

```text
cnt--
```

注意：

```text
只有两个集合原本不同，合并后连通块数量才减少。
```

## 离线反向加边 / 加点

并查集擅长合并，不擅长删除。

如果题目要求：

```text
不断删除点或删除边，并查询连通块数量。
```

可以尝试反过来处理：

```text
删除问题反向看，就是加点或加边。
```

P1197 就是典型例子：

```text
正向：不断摧毁星球。
反向：从最后状态开始，逐个恢复星球。
```

## 本阶段常见错误

- `find` 没有路径压缩，导致超时；
- 合并时没有先找根；
- 连通块数量每次操作都减一，没判断是否真的合并；
- 无向边只加入一次但后续遍历写错；
- 删除类问题强行在线删除，导致并查集不好维护；
- 多组数据忘记重新初始化；
- 数组开小。

## 二倍扩展域

如果每个元素只有两种状态：

```text
和我同类
和我相反
```

可以给每个点 `x` 建两个编号：

```text
x：x 本身
x + n：x 的反面
```

如果 `x` 和 `y` 相反，则合并：

```text
x 与 y+n
x+n 与 y
```

如果 `x` 和 `y` 相同，则合并：

```text
x 与 y
x+n 与 y+n
```

冲突判断：

```text
如果 find(x) == find(x+n)，说明 x 同时与自己的反面相同，矛盾。
```

## 三倍扩展域

食物链这类题有三种关系：

```text
同类
吃
被吃
```

可以把每个动物拆成三类状态：

```text
x
x + n
x + 2n
```

一种常见定义是：

```text
x：x 所属种类
x+n：x 吃的种类
x+2n：吃 x 的种类
```

然后根据题目语句合并相应状态。

## P1955：等式与不等式约束

P1955 的核心流程：

```text
1. 所有变量编号可能很大，先离散化；
2. 先处理所有等式 x = y，把它们合并；
3. 再处理所有不等式 x != y；
4. 如果某个不等式两边已经在同一集合，矛盾。
```

这类题体现了并查集的一个重要思想：

```text
能合并的确定关系先合并，再检查禁止关系是否被破坏。
```

## 本阶段常见错误

- 二倍扩展域中反面编号写错；
- 没有判断 `x` 与 `x+n` 是否冲突；
- 食物链三倍拆点时关系定义前后不一致；
- P1525 中边要按怨气值从大到小处理；
- P1955 中变量值可到 `1e9`，忘记离散化；
- 等式和不等式一起处理，导致错误判断；
- 忘记多组数据重新初始化。

## 1. 带权并查集维护什么？

普通并查集维护：

```text
x 与 y 是否属于同一集合。
```

带权并查集维护：

```text
x 与 y 属于同一集合时，x 和 y 之间的相对关系。
```

通常定义：

```cpp
int fa[N];
long long d[N];
```

其中：

```text
fa[x]：x 的父节点
d[x]：x 到 fa[x] 的权值
```

路径压缩后：

```text
d[x]：x 到根节点的权值
```

---

## 2. find 中如何更新权值？

核心模板：

```cpp
int find(int x) {
    if (fa[x] == x) return x;

    int oldFather = fa[x];

    fa[x] = find(fa[x]);

    d[x] += d[oldFather];

    return fa[x];
}
```

这段代码的含义是：

```text
原来：
x -> oldFather -> root

路径压缩后：
x -> root

因此：
d[x] = x 到 oldFather 的权值 + oldFather 到 root 的权值
```

如果维护的是模意义关系，则写成：

```cpp
d[x] = (d[x] + d[oldFather]) % mod;
```

如果维护的是异或关系，则写成：

```cpp
d[x] ^= d[oldFather];
```

---

## 3. 合并时如何推导权值？

假设我们知道一个约束：

```text
relation(x, y) = w
```

也就是：

```text
x 到 y 的关系值为 w
```

并且：

```text
rx = find(x)
ry = find(y)
```

路径压缩后：

```text
d[x] = x 到 rx 的关系
d[y] = y 到 ry 的关系
```

如果要把 `rx` 接到 `ry` 下，需要设置：

```text
d[rx] = ?
```

常见公式为：

```cpp
fa[rx] = ry;
d[rx] = d[y] - d[x] + w;
```

这表示：

```text
x 到 y 的关系 = d[x] + d[rx] - d[y] = w
```

所以：

```text
d[rx] = d[y] - d[x] + w
```

如果是模关系：

```cpp
d[rx] = (d[y] - d[x] + w + mod) % mod;
```

如果是异或关系：

```cpp
d[rx] = d[x] ^ d[y] ^ w;
```

注意：

```text
公式不是死背，要根据你定义的 d[x] 方向推导。
```

## 4. 距离型带权并查集：P1196 银河英雄传说

P1196 要维护：

```text
两艘战舰是否在同一列；
如果在同一列，询问它们之间有多少战舰。
```

需要维护：

```text
fa[x]：x 所在队列的代表；
d[x]：x 到队列根的距离；
sz[root]：当前队列大小。
```

合并时：

```text
把 i 所在队列接到 j 所在队列尾部。
```

如果 `ri` 接到 `rj`，那么：

```text
d[ri] = sz[rj]
sz[rj] += sz[ri]
```

查询时：

```text
如果 find(i) != find(j)，输出 -1；
否则输出 abs(d[i] - d[j]) - 1。
```

## 5. 模 3 关系型带权并查集：P2024 食物链

可以定义：

```text
d[x] 表示 x 与根节点的关系，取值 mod 3。
```

一种常用定义：

```text
0：x 与根同类
1：x 吃根
2：x 被根吃
```

对于语句：

```text
1 X Y：X 和 Y 同类
2 X Y：X 吃 Y
```

把它转成 `X` 与 `Y` 的关系值 `w`。

例如定义：

```text
relation(x, y) = d[x] - d[y] (mod 3)
```

则：

```text
X 和 Y 同类：w = 0
X 吃 Y：w = 1
```

如果 `X` 和 `Y` 已经在同一集合，就检查：

```text
(d[x] - d[y] + 3) % 3 == w
```

否则合并，并设置根的关系值。

## 6. 奇偶关系型带权并查集：P5937 Parity Game

P5937 的约束是：

```text
[l, r] 中 1 的个数是 odd / even。
```

设：

```text
pre[i] = 前 i 个数中 1 的个数 mod 2
```

那么：

```text
[l, r] 中 1 的个数奇偶
= pre[r] ^ pre[l-1]
```

于是每条语句就变成：

```text
pre[l-1] 与 pre[r] 的异或关系。
```

用带权并查集维护：

```text
d[x] 表示 x 到根的异或值。
```

合并公式：

```cpp
d[rx] = d[x] ^ d[y] ^ w;
```

其中 `w` 为当前约束的奇偶：

```text
even -> 0
odd -> 1
```

由于 `n` 可达 `1e9`，而询问只有 `m` 个，所以必须离散化 `l-1` 和 `r`。

## 7. 前缀和差值型带权并查集：P2294 狡猾的商人

这类题通常给出若干区间和：

```text
a[l] + a[l+1] + ... + a[r] = s
```

设前缀和：

```text
pre[i] = a[1] + a[2] + ... + a[i]
```

则：

```text
pre[r] - pre[l-1] = s
```

这就变成两个点之间的差值约束：

```text
pre[r] 与 pre[l-1] 的差值为 s。
```

用带权并查集维护：

```text
d[x] = pre[x] - pre[root]
```

如果已经在同一集合，则检查：

```text
d[r] - d[l-1] == s
```

如果不在同一集合，则合并并设置根的权值。

---

## 本阶段常见错误

- `d[x]` 到底是“x 到父亲”还是“x 到根”没有统一；
- `find` 里先改了 `fa[x]`，导致旧父亲权值丢失；
- 合并公式死背，没有根据方向推导；
- 模关系忘记加 mod 防止负数；
- 异或关系误用加减法；
- 查询前没有先 `find(x)` 和 `find(y)`，导致 `d[x]` 不是到根的权值；
- 离散化时漏掉 `l-1`；
- P1196 中队列大小 `sz[root]` 没有维护；
- P2024 中 `X > N` 或 `Y > N`、`X 吃 X` 这些假话条件漏判；
- 多组数据忘记清空 `fa`、`d`、`sz`。

## 可撤销并查集

普通路径压缩会直接改父亲，难以撤销。

可撤销并查集通常：

```text
不使用路径压缩；
使用按秩合并或按大小合并；
把每次合并前被修改的信息压栈；
回滚时从栈中恢复。
```

适合：

```text
线段树分治维护动态连通性；
离线处理加边删边；
需要撤销历史操作。
```

## 可持久化并查集

可持久化并查集要求：

```text
能够回到某一次历史操作后的状态。
```

P3402 中有三种操作：

```text
合并两个集合；
回到第 k 次操作后的状态；
查询两个元素是否在同一集合。
```

常用实现：

```text
可持久化数组 / 主席树维护 fa 和 dep。
```

这是并查集的高阶扩展，建议在普通并查集、按秩合并、线段树基础较扎实后再学。

## 并查集维护集合信息

除了连通性，并查集还可以在根节点维护集合信息：

```text
集合大小；
集合最大值 / 最小值；
集合成员数量；
集合边数；
集合是否有环；
集合权值和；
集合内特殊点个数。
```

常见写法：

```cpp
if (merge(x, y)) {
    sz[root] += sz[other];
    sum[root] += sum[other];
}
```

注意：

```text
只有根节点上的集合信息是可靠的。
```

每次查询前应该先 `find(x)`。

## 本阶段常见错误

- 可撤销并查集中仍然使用路径压缩，导致无法回滚；
- 合并时没有记录足够多的历史信息；
- 回滚时集合大小、连通块数量没有恢复；
- 可持久化并查集中版本编号处理错误；
- 根节点集合信息没有在合并时同步；
- 离线删除题没有正确反向处理；
- 并查集用于有向图强连通判断，这是不适合的。

# 每道题的复盘模板

建议每做完一道并查集题，都写下面几个问题。

## 1. 本题并查集维护什么？

例如：

```text
维护两个点是否连通。
```

或者：

```text
维护敌友关系。
```

或者：

```text
维护两个前缀和之间的差值。
```

## 2. 这是普通并查集、扩展域，还是带权并查集？

普通并查集：

```text
只需要判断是否在同一集合。
```

扩展域：

```text
每个元素拆成多个状态。
```

带权并查集：

```text
需要维护同集合内两个元素的相对关系。
```

## 3. 如果是带权并查集，`d[x]` 定义是什么？

一定要写清楚：

```text
d[x] 表示 x 到父节点的关系。
```

或者：

```text
路径压缩后，d[x] 表示 x 到根节点的关系。
```

还要明确关系类型：

```text
距离和
模 3
异或
差值
```

## 4. 新约束如何转化？

例如：

```text
[l, r] 的和为 s
=> pre[r] - pre[l-1] = s
```

或者：

```text
[l, r] 中 1 的个数为奇数
=> pre[r] ^ pre[l-1] = 1
```

或者：

```text
x 吃 y
=> relation(x, y) = 1 (mod 3)
```

## 5. 合并公式如何推导？

不要只背模板。要根据 `d[x]` 的定义推导：

```text
已知 relation(x, y) = w
已知 d[x] 和 d[y]
要求 fa[rootX] = rootY 时 d[rootX] 的值
```

## 6. 什么时候判断矛盾？

如果 `x` 和 `y` 已经在同一集合：

```text
检查当前 d[x] 与 d[y] 推出的关系是否等于新约束。
```

如果不在同一集合：

```text
合并，并设置根节点之间的权值。
```

## 7. 是否需要离散化？

如果元素编号很大，例如：

```text
1 <= x <= 1e9
```

而操作数量不大，就应考虑离散化。

P1955、P5937 都体现了这一点。

# 带权并查集四种典型模型

## 1. 距离模型

维护：

```text
d[x] = x 到根的距离
```

代表题：

```text
P1196
UVA1329
```

常见查询：

```text
同集合内两个点距离 = abs(d[x] - d[y])
```

## 2. 奇偶模型

维护：

```text
d[x] = x 到根的奇偶关系
```

取值：

```text
0 / 1
```

合并常用：

```text
d[rootX] = d[x] ^ d[y] ^ w
```

代表题：

```text
P5937
```

## 3. 模关系模型

维护：

```text
d[x] = x 到根的关系 mod k
```

典型为模 3：

```text
0：同类
1：吃
2：被吃
```

代表题：

```text
P2024
POJ1182
```

## 4. 差值约束模型

维护：

```text
d[x] = value[x] - value[root]
```

常用于：

```text
前缀和差值
变量差值
区间和约束
```

代表题：

```text
P2294
```

# 并查集与其他算法的选择

## 更适合并查集的情况

- 只关心连通性；
- 只进行合并，不进行在线删除；
- 要维护若干等价类；
- 关系具有传递性；
- 可以离线把删除转成添加；
- 可以用扩展域表示状态；
- 可以用带权关系维护相对值。

## 不适合普通并查集的情况

- 需要在线删除边或点；
- 需要维护有向图可达性；
- 需要查询两点之间具体路径；
- 需要动态维护最短路；
- 关系不具有传递性；
- 合并后还需要拆开。

这些情况可能要考虑：

```text
动态树
LCT
线段树分治 + 可撤销并查集
BFS / DFS
最短路
拓扑排序
强连通分量
```

## 扩展域和带权并查集怎么选？

如果状态种类少，且关系容易拆点：

```text
优先扩展域，直观。
```

例如：

```text
敌友关系
二分关系
三类食物链
```

如果关系更像数值差、距离、奇偶、模关系：

```text
优先带权并查集。
```

例如：

```text
距离差
前缀和差
区间奇偶
模 3 捕食关系
```

很多题两种写法都能做，比如：

```text
P2024 食物链
P5937 Parity Game
```

但带权并查集更能锻炼“相对关系建模”。

# 最终掌握标准

学完并查集，尤其是带权并查集后，看到一道题应该能快速判断：

1. 题目是否只关心集合关系？
2. 关系是否具有传递性？
3. 是普通连通性，还是有敌友/种类/差值关系？
4. 是否能用扩展域拆点？
5. 是否需要维护同集合内两个元素的相对值？
6. 如果是带权并查集，`d[x]` 的定义是什么？
7. `find` 路径压缩时权值如何更新？
8. 合并两个集合时，根节点的权值如何推导？
9. 查询时应该比较 `d[x] - d[y]`、`d[x] ^ d[y]`，还是 `(d[x]-d[y]) mod k`？
10. 是否存在矛盾判断？
11. 是否需要离散化？
12. 是否有删除操作，需要离线反向处理或可撤销并查集？
13. 是否需要维护集合大小、距离、边数等根节点信息？

如果这些问题都能回答出来，说明你已经不只是会写普通并查集模板，而是能用并查集处理连通性、逻辑关系和带权约束问题。
