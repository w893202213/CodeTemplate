# Manacher 算法到底解决什么问题？

Manacher 算法，也常被称为“马拉车算法”，主要用于在线性时间内处理回文子串问题。

它最经典的用途是：

```text
给定一个字符串 s，求 s 中最长回文子串的长度。
```

暴力枚举所有子串再判断回文，复杂度通常是：

```text
O(n^3)
```

中心扩展法枚举每个中心向两边扩展，复杂度是：

```text
O(n^2)
```

Manacher 利用已经求出的回文半径，避免重复扩展，可以做到：

```text
O(n)
```

它常用于：

- 最长回文子串；
- 统计回文子串数量；
- 统计奇数长度回文串；
- 判断每个位置能向左右扩展多远；
- 求以每个位置为中心的最长回文半径；
- 双回文串、回文覆盖、回文匹配等进阶问题；
- 与前缀和、差分、KMP、DP 结合。

---

# Manacher 与中心扩展法的关系

中心扩展法是：

```text
枚举每个中心，然后向左右扩展。
```

Manacher 也是围绕“中心”展开，但它维护一个当前能到达最右端的回文区间：

```text
[l, r]
```

当新的中心 `i` 落在这个区间内时，可以利用对称点的信息，直接得到一个初始半径。

也就是说：

```text
Manacher = 利用对称性优化后的中心扩展法。
```

这点和 Z 函数很像：

```text
Z 函数维护最右匹配区间；
Manacher 维护最右回文区间。
```

## 统一分隔符写法

为了统一处理奇数长度和偶数长度回文，可以在字符之间插入分隔符。

例如：

```text
原串：abba
处理后：#a#b#b#a#
```

这样原串中的偶数回文：

```text
abba
```

在新串中也会变成以某个位置为中心的奇数回文。

常见处理方式：

```cpp
string transform(const string& s) {
    string t = "#";
    for (char c : s) {
        t += c;
        t += "#";
    }
    return t;
}
```

## Manacher 基础模板

```cpp
vector<int> manacher(const string& s) {
    string t = "#";
    for (char c : s) {
        t += c;
        t += "#";
    }

    int n = t.size();
    vector<int> p(n, 0);

    int center = 0;
    int right = 0;

    for (int i = 0; i < n; i++) {
        int mirror = 2 * center - i;

        if (i < right) {
            p[i] = min(right - i, p[mirror]);
        }

        while (i - p[i] - 1 >= 0 &&
               i + p[i] + 1 < n &&
               t[i - p[i] - 1] == t[i + p[i] + 1]) {
            p[i]++;
        }

        if (i + p[i] > right) {
            center = i;
            right = i + p[i];
        }
    }

    return p;
}
```

## 如何由 `p[i]` 得到答案？

在分隔符版本中：

```text
p[i] = 处理串中以 i 为中心的最长回文半径
```

原串中的最长回文长度通常就是：

```cpp
ans = max(p[i])
```

例如：

```text
原串 abba
处理后 #a#b#b#a#
中心在中间的 #，p[i] = 4
最长回文长度 = 4
```

## 不插入分隔符的 d1 / d2 写法

另一种常见写法是分别求：

```text
d1[i]：以 i 为中心的奇数长度回文半径
d2[i]：以 i 和 i-1 之间为中心的偶数长度回文半径
```

含义：

```text
d1[i] = k 表示 s[i-k+1...i+k-1] 是回文串
d2[i] = k 表示 s[i-k...i+k-1] 是回文串
```

最长回文长度：

```text
奇数回文：2 * d1[i] - 1
偶数回文：2 * d2[i]
```

## 本阶段常见错误

- 处理串长度和原串长度没有对应清楚；
- `right` 是最右覆盖位置，还是最右位置的后一位，模板中前后不统一；
- `mirror = 2 * center - i` 越界；
- `i < right` 和 `i <= right` 混用；
- 分隔符版本中把 `p[i]` 错当成处理串长度；
- 忘记原串中最长回文长度可以直接取 `max(p[i])`；
- 字符串很长时使用 `string +=` 可能较慢，可预先 `reserve`。

| 4 | 洛谷 P1872 回文串计数 | https://www.luogu.com.cn/problem/P1872 | 回文计数方向题，可作为扩展练习 |

## 如何统计回文子串数量？

如果使用 `d1 / d2` 写法：

```text
d1[i] 表示以 i 为中心的奇数回文半径数量
d2[i] 表示以 i 和 i-1 之间为中心的偶数回文半径数量
```

那么总回文子串数量就是：

```cpp
long long ans = 0;

for (int i = 0; i < n; i++) {
    ans += d1[i];
    ans += d2[i];
}
```

原因是：

```text
d1[i] = k，说明以 i 为中心有 k 个奇数回文：
长度分别为 1, 3, 5, ..., 2k-1

d2[i] = k，说明以 i 和 i-1 之间为中心有 k 个偶数回文：
长度分别为 2, 4, 6, ..., 2k
```

## 拉拉队排练：只统计奇数长度回文

P1659 的核心是：

```text
只考虑奇数长度回文串。
```

Manacher 可以得到每个中心的最大奇数回文长度。

如果某个中心存在长度为 `len` 的奇数回文，那么它也一定存在：

```text
len - 2
len - 4
...
1
```

这些更短的奇数回文。

所以可以按长度统计出现次数，再从大到小选取前 `k` 个长度贡献。

## 反对称串与 Manacher 思想

P3501 不完全是普通回文串，而是反对称关系。

普通回文要求：

```text
s[l] == s[r]
```

反对称可以理解为：

```text
s[l] != s[r]
```

并且通常只会出现偶数长度。

它仍然可以借鉴 Manacher 的“中心扩展 + 复用半径”思想，只是扩展条件发生变化。

## 本阶段常见错误

- 统计回文数量时把最长长度累加，而不是半径数量；
- 奇数回文长度与半径换算错误；
- P1659 中没有注意只取奇数长度；
- 按长度统计时没有把长回文对子回文的贡献向下传递；
- 答案很大时没有使用 `long long`；
- 模数题忘记快速幂或取模。

## 双回文串的核心思想

P4555 要求找到一个最长子串 `T`，使它可以拆成两部分：

```text
T = X + Y
```

其中：

```text
X 是回文串
Y 是回文串
```

做法思路：

1. 用 Manacher 求出所有中心的最大回文半径；
2. 转换得到：
   - `left[i]`：以 `i` 为左端点的最长回文长度；
   - `right[i]`：以 `i` 为右端点的最长回文长度；
3. 枚举切分点；
4. 用左右两侧的回文长度更新答案。

关键点：

```text
Manacher 给的是“中心信息”，题目需要的是“端点信息”。
```

因此中等题常常要做一次信息转换。

## 力扣 1960：两个不重叠奇数回文最大乘积

题目要求两个不重叠的奇数长度回文子串，使长度乘积最大。

常见做法：

```text
1. 用 Manacher 求每个中心的奇数回文半径；
2. 求 leftBest[i]：以 i 及其左侧为范围的最大奇数回文长度；
3. 求 rightBest[i]：以 i 及其右侧为范围的最大奇数回文长度；
4. 枚举切分点 i，答案为 leftBest[i] * rightBest[i + 1]。
```

注意：

```text
必须保证两个回文子串不重叠。
```

## 最短回文串与最长回文前缀

力扣 214 要在字符串前面添加字符，使整体变成回文串。

核心是找：

```text
原串的最长回文前缀。
```

如果最长回文前缀是：

```text
s[0...k]
```

那么只需要把剩余部分：

```text
s[k+1...n-1]
```

反转后加到前面。

这个题可以用 KMP、Z 函数，也可以用 Manacher 判断回文前缀。

## 本阶段常见错误

- Manacher 半径是中心信息，不能直接当作端点答案；
- 转换左端点/右端点信息时下标错位；
- 分隔符版本和原串下标映射错误；
- 双回文串题中两部分必须都非空；
- 力扣 1960 中两个回文必须不重叠；
- 最短回文串只允许在前面添加字符，不能在后面添加。

## 回文匹配：Manacher + KMP

P6216 的典型思路是：

```text
1. 用 KMP 找出一个模式串在主串中的所有出现位置；
2. 用 Manacher 求出主串中所有回文覆盖范围；
3. 统计每个匹配位置被多少个回文串覆盖；
4. 用前缀和或差分优化贡献计算。
```

这类题的特点是：

```text
KMP 负责模式匹配；
Manacher 负责回文结构；
前缀和/差分负责批量统计贡献。
```

## Manacher 与回文自动机的区别

Manacher 适合：

```text
一次性处理一个固定字符串中的所有回文半径。
```

回文自动机 PAM 适合：

```text
动态加入字符，同时维护不同回文子串。
```

如果题目问：

```text
最长回文子串
回文半径
每个中心扩展多远
```

优先考虑 Manacher。

如果题目问：

```text
不同回文子串数量
每个回文子串出现次数
动态添加字符后的回文信息
```

更适合回文自动机。

## Manacher 与 KMP / Z 函数的联系

这三个算法都利用了“已知信息复用”：

```text
KMP：复用前缀函数，失配时跳转。
Z 函数：复用最右匹配区间，求每个后缀与前缀的 LCP。
Manacher：复用最右回文区间，求每个中心的回文半径。
```

如果你已经学过 Z 函数，会发现 Manacher 的核心结构非常相似。

## 本阶段常见错误

- 把 Manacher 当成万能回文算法，忽略 PAM 更适合统计不同回文串；
- KMP 和 Manacher 的下标体系没有统一；
- 前缀和/差分统计贡献时边界错一位；
- 处理串中加入分隔符后，原串位置映射错误；
- 复杂题中没有先明确“中心信息要转成什么信息”。

# 每道题的复盘模板

建议每做完一道 Manacher 题，都写下面几个问题。

## 1. 本题需要 Manacher 求什么？

例如：

```text
本题需要求最长回文子串长度。
```

或者：

```text
本题需要求每个中心的最大回文半径。
```

或者：

```text
本题需要统计所有奇数长度回文串。
```

## 2. 使用分隔符版本，还是 d1 / d2 版本？

例如：

```text
本题奇偶回文都要处理，所以使用分隔符版本。
```

或者：

```text
本题只关心奇数长度回文，所以使用 d1 数组更直接。
```

## 3. 半径数组如何转化成题目需要的信息？

例如：

```text
p[i] 是中心信息，但本题需要以每个位置为右端点的最长回文长度，因此要做端点转换。
```

## 4. 是否需要统计所有较短回文？

如果一个中心存在最大长度为 `len` 的回文，它通常也包含：

```text
len - 2
len - 4
...
1
```

这些较短回文。

## 5. 是否需要前缀和、差分或 DP？

例如：

```text
本题需要批量统计回文覆盖次数，因此需要对 Manacher 半径产生的贡献做差分。
```

## 6. 是否应该改用回文自动机？

如果题目要求：

```text
不同回文子串数量
每个不同回文串出现次数
动态添加字符
```

就要考虑回文自动机 PAM。

# Manacher 与其他回文算法的选择

## 更适合 Manacher 的情况

- 求最长回文子串；
- 求每个中心的回文半径；
- 统计回文子串总数量；
- 处理固定字符串；
- 需要 O(n) 处理所有中心；
- 题目需要大量回文半径信息。

## 更适合中心扩展法的情况

- 字符串长度较小；
- 只做面试题，不追求最优复杂度；
- 代码简单优先；
- `O(n^2)` 可以通过。

## 更适合动态规划的情况

- 需要判断任意子串是否为回文；
- 后续还有区间 DP；
- 字符串长度不大；
- 需要直接访问 `isPalindrome[l][r]`。

## 更适合 KMP / Z 函数的情况

- 需要求最长回文前缀或后缀；
- 问题可以转成 `s + '#' + reverse(s)` 的前后缀匹配；
- 只关心某一类边界匹配，不需要所有中心半径。

## 更适合回文自动机 PAM 的情况

- 统计不同回文子串数量；
- 统计每个不同回文串出现次数；
- 动态加入字符；
- 需要维护回文串之间的 fail 关系。

# 三类 Manacher 问题的区别

## 1. 模板最长回文类

目标：

```text
求最长回文子串长度或具体子串。
```

代表题：

```text
P3805
LeetCode 5
```

## 2. 回文计数类

目标：

```text
统计所有回文子串，或统计某种长度/奇偶限制下的回文串。
```

代表题：

```text
LeetCode 647
P1659
P3501
P1872
```

## 3. 回文边界与综合类

目标：

```text
把中心半径转成端点信息、覆盖信息或切分信息。
```

代表题：

```text
P4555
LeetCode 1960
P6216
P5446
```

# 最终掌握标准

学完 Manacher 后，看到一道题应该能快速判断：

1. 题目是否围绕回文子串？
2. 只需要最长回文，还是需要所有中心的半径？
3. 是否需要同时处理奇数回文和偶数回文？
4. 分隔符版本与 d1/d2 版本哪个更合适？
5. 半径数组能否直接得到答案，还是需要转换成端点信息？
6. 是否需要统计所有较短回文的贡献？
7. 是否需要结合前缀和、差分、KMP 或 DP？
8. 是否更适合使用回文自动机 PAM？

如果这些问题都能回答出来，说明你已经不只是会背 Manacher 模板，而是能真正用回文半径建模。
