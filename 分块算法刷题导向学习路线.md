# 分块算法到底解决什么问题？

分块算法的核心是：

```text
把一个规模为 n 的问题拆成若干块。
对于完整块，用预处理或标记快速处理；
对于零散部分，直接暴力处理。
```

典型块长通常取：

```text
B ≈ sqrt(n)
```

因此很多操作可以做到：

```text
O(sqrt(n))
```

分块的特点：

- 思想灵活；
- 代码通常比线段树短；
- 能维护一些线段树不方便维护的信息；
- 渐近复杂度通常不如线段树、树状数组；
- 适合数据范围中等、操作复杂、难以维护 lazy 的题。

常见类型：

1. 数列分块；
2. 块内排序；
3. 块内懒标记；
4. 根号分治；
5. 莫队算法；
6. 区间众数分块；
7. 弹跳类分块；
8. 块状链表。

## 基础模板：区间加 + 区间求和

```cpp
const int N = 100000 + 10;

int n, B, num;
long long a[N], sum[N], tag[N];
int bel[N], L[N], R[N];

void build() {
    B = sqrt(n);
    num = (n + B - 1) / B;

    for (int i = 1; i <= num; i++) {
        L[i] = (i - 1) * B + 1;
        R[i] = min(i * B, n);
    }

    for (int i = 1; i <= n; i++) {
        bel[i] = (i - 1) / B + 1;
        sum[bel[i]] += a[i];
    }
}

void rangeAdd(int l, int r, long long v) {
    int x = bel[l], y = bel[r];

    if (x == y) {
        for (int i = l; i <= r; i++) {
            a[i] += v;
            sum[x] += v;
        }
        return;
    }

    for (int i = l; i <= R[x]; i++) {
        a[i] += v;
        sum[x] += v;
    }

    for (int i = L[y]; i <= r; i++) {
        a[i] += v;
        sum[y] += v;
    }

    for (int i = x + 1; i <= y - 1; i++) {
        tag[i] += v;
        sum[i] += v * (R[i] - L[i] + 1);
    }
}

long long rangeQuery(int l, int r) {
    int x = bel[l], y = bel[r];
    long long ans = 0;

    if (x == y) {
        for (int i = l; i <= r; i++) {
            ans += a[i] + tag[x];
        }
        return ans;
    }

    for (int i = l; i <= R[x]; i++) {
        ans += a[i] + tag[x];
    }

    for (int i = L[y]; i <= r; i++) {
        ans += a[i] + tag[y];
    }

    for (int i = x + 1; i <= y - 1; i++) {
        ans += sum[i];
    }

    return ans;
}
```

## 本阶段必须理解

### 1. 分块处理区间的三段结构

一个区间 `[l, r]` 通常被拆成：

```text
左边角块：暴力处理
中间完整块：整块处理
右边角块：暴力处理
```

### 2. 为什么复杂度是 O(sqrt n)

如果块长取 `B = sqrt(n)`：

```text
边角块最多处理 2B 个元素；
中间完整块最多有 n / B 个。
```

所以单次操作复杂度大致是：

```text
O(B + n / B)
```

当：

```text
B = sqrt(n)
```

复杂度最优，大约为：

```text
O(sqrt(n))
```

## 本阶段常见错误

- 块编号从 0 和从 1 开始混用；
- 最后一块右端点没有取 `min(i * B, n)`；
- 更新边角块时忘记维护块和；
- 查询边角块时忘记加懒标记；
- 完整块加 tag 后没有同步维护 `sum`；
- 同一个块内的区间没有单独处理。

## 块内排序思想

对于每个块，额外维护一个排序后的数组：

```cpp
vector<int> sortedBlock[id];
```

查询区间内小于 `x` 的个数时：

- 边角块：暴力扫；
- 完整块：在有序数组中二分。

例如完整块内查询：

```cpp
cnt += lower_bound(v.begin(), v.end(), x - tag[id]) - v.begin();
```

如果是查询大于等于 `x` 的个数：

```cpp
cnt += v.end() - lower_bound(v.begin(), v.end(), x - tag[id]);
```

## 块内重构

当边角块被暴力修改后，该块的原始数组改变了。

因此要重新排序这个块：

```cpp
void rebuild(int id) {
    vec[id].clear();
    for (int i = L[id]; i <= R[id]; i++) {
        vec[id].push_back(a[i]);
    }
    sort(vec[id].begin(), vec[id].end());
}
```

## 弹飞绵羊类分块

P3203 的思想不是区间加，而是维护：

```text
jump[i]：从 i 开始跳，跳出当前块后到达的位置；
step[i]：从 i 开始跳，跳出当前块需要的步数。
```

单点修改后，只需要重构该位置所在的块。

查询时：

```text
不断整块跳，直到跳出整个数组。
```

## 本阶段常见错误

- 块内排序数组没有重构；
- 二分时忘记减去块懒标记；
- 修改边角块时没有先下放 tag；
- 区间开方题中没有利用“数值很快变小”的性质剪枝；
- 弹飞绵羊中 `jump` 和 `step` 更新方向写反；
- 块状链表题中没有定期重构，导致块过大。

## 根号分治

根号分治通常适合这种题：

```text
参数 x 有大有小。
当 x 很小时，预处理；
当 x 很大时，暴力次数本来就少。
```

以 P3396 为例，可以按模数大小分治：

- 小模数：预处理答案；
- 大模数：每次查询暴力枚举，复杂度也不会太高。

核心判断：

```text
如果某个参数 <= sqrt(n)，走预处理；
如果某个参数 > sqrt(n)，走暴力。
```

## 普通莫队

莫队用于离线区间查询。

基本流程：

```text
1. 把所有询问按左端点所在块排序；
2. 同一块内再按右端点排序；
3. 用两个指针 L、R 维护当前区间；
4. 每次移动指针时增删一个元素；
5. 维护当前答案。
```

普通莫队适合：

```text
已知如何 O(1) 或 O(log n) 添加/删除一个元素，并维护答案。
```

## 莫队的典型 add / del 思维

以 P2709 为例，答案是：

```text
sum(cnt[x]^2)
```

添加一个数 `x`：

```cpp
ans -= cnt[x] * cnt[x];
cnt[x]++;
ans += cnt[x] * cnt[x];
```

删除一个数 `x`：

```cpp
ans -= cnt[x] * cnt[x];
cnt[x]--;
ans += cnt[x] * cnt[x];
```

## 本阶段常见错误

- 莫队排序规则写错；
- 指针移动顺序导致区间维护错误；
- add 和 del 没有保持互逆；
- 离散化遗漏，导致计数数组越界；
- 答案涉及组合数时没有先减旧贡献再加新贡献；
- 根号分治中阈值设置不合理，但一般 `sqrt(n)` 附近即可。

## 带修改莫队

普通莫队维护两个指针：

```text
L, R
```

带修改莫队多了一个时间维度：

```text
L, R, T
```

每个询问不仅有区间 `[l, r]`，还对应前面发生了多少次修改。

排序时通常按三维分块排序：

```text
左端点块
右端点块
时间块
```

维护时需要支持：

```text
加入一个位置
删除一个位置
应用一次修改
撤销一次修改
```

## 树上莫队

树上路径询问不能直接用数组区间表示，因此通常先做：

```text
DFS 序 / 欧拉序
```

把树上路径转成序列上的若干点出现关系。

常见配套知识：

- LCA；
- 欧拉序；
- 点出现次数奇偶性；
- 莫队排序；
- add / del 维护答案。

## 区间众数分块

区间众数难点在于：

```text
众数不具备简单的区间合并性质。
```

线段树很难直接维护在线区间众数。

分块常见做法：

```text
1. 离散化所有数；
2. 预处理每个块前缀中各数出现次数；
3. 预处理整块区间的众数；
4. 查询时用整块答案作为候选；
5. 边角块暴力枚举候选，并用前缀计数计算出现次数。
```

## 本阶段常见错误

- 带修改莫队中修改没有判断位置是否在当前区间内；
- 撤销修改时没有恢复原值；
- 树上莫队中 LCA 点重复或遗漏；
- 欧拉序中的点出现两次，奇偶切换逻辑写错；
- 区间众数分块中候选值统计不完整；
- 预处理数组空间过大，导致内存爆炸。

# 每道题的复盘模板

建议每做完一道分块题，都写下面几个问题。

## 1. 本题为什么可以分块？

例如：

```text
本题需要区间修改和区间查询，完整块可以用懒标记处理，边角块可以暴力处理。
```

或者：

```text
本题询问可以离线排序，左右端点移动时能 O(1) 增删元素，因此可以使用莫队。
```

## 2. 块长怎么取？

常见答案：

```text
块长取 sqrt(n)。
```

或者：

```text
莫队中块长一般取 sqrt(n)，带修莫队中块长可能取 n^(2/3) 附近。
```

## 3. 完整块维护什么？

例如：

```text
完整块维护区间和 sum 和加法懒标记 tag。
```

或者：

```text
完整块维护块内排序数组，用于二分统计小于 x 的数量。
```

## 4. 边角块怎么处理？

例如：

```text
边角块直接暴力扫，并在修改后重构该块。
```

## 5. 是否需要重构？

如果块内维护了有序数组、跳跃信息、众数候选等内容，修改后往往需要重构。

# 分块与其他算法的选择

## 更适合分块的情况

- 区间操作较复杂，线段树 lazy 不好维护；
- 信息不容易合并；
- 数据范围允许 `O(n sqrt n)` 或 `O(q sqrt n)`；
- 需要在线处理区间众数、跳跃、块状链表；
- 离线区间询问可以通过莫队维护；
- 参数有大有小，适合根号分治。

## 更适合线段树的情况

- 区间信息可以合并；
- 区间修改标记清晰；
- 数据范围较大，需要 `O(log n)`；
- 需要严格在线且操作标准。

## 更适合树状数组的情况

- 单点修改 + 区间求和；
- 区间修改 + 单点查询；
- 区间修改 + 区间求和；
- 统计前缀和、逆序对、排名。

## 更适合莫队的情况

- 没有修改，或修改可以离线处理；
- 询问都是区间查询；
- add / del 操作能快速维护答案；
- 查询顺序不影响最终输出，可以离线排序。

# 三类常见分块思想

## 1. 序列分块

维护数组区间。

代表题：

```text
LOJ 6277
LOJ 6278
LOJ 6280
```

# 最终掌握标准

学完分块后，看到一道题应该能快速判断：

1. 是否能把序列按块划分？
2. 完整块能否快速处理？
3. 边角块能否暴力处理？
4. 块内需要维护和、标记、排序数组，还是跳跃信息？
5. 修改后是否需要重构块？
6. 复杂度是否是 `O(sqrt n)` 或 `O(n sqrt n)` 级别？
7. 是否可以用莫队离线处理区间询问？
8. 是否存在“小参数预处理，大参数暴力”的根号分治结构？
9. 是否比线段树、树状数组更适合当前问题？

如果这些问题都能回答出来，说明你已经不是只会套分块模板，而是能把分块作为一种“优雅暴力”的建模方法。
