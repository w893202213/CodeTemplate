# 后缀数组到底解决什么问题？

后缀数组用于处理字符串的所有后缀。

给定字符串：

```text
s = banana
```

它的所有后缀是：

```text
1: banana
2: anana
3: nana
4: ana
5: na
6: a
```

把这些后缀按字典序排序后，得到后缀起始位置的排列：

```text
sa = [6, 4, 2, 1, 5, 3]
```

这就是后缀数组。

后缀数组最常见的三个数组：

```text
sa[i]：排名第 i 小的后缀的起始位置。
rk[i]：起始位置为 i 的后缀的排名。
height[i]：sa[i] 与 sa[i-1] 这两个相邻后缀的最长公共前缀长度。
```

后缀数组常用于：

- 后缀排序；
- 子串查找；
- 统计本质不同子串数量；
- 求最长重复子串；
- 求出现至少 k 次的最长子串；
- 求两个字符串最长公共子串；
- 求多个字符串最长公共子串；
- 统计所有后缀两两 LCP 贡献；
- 字典序相关问题；
- 与 RMQ、单调栈、并查集结合。

# 后缀数组学习重点

学习后缀数组时，不要只停留在“会输出 sa”。

真正刷题时最核心的是：

```text
height 数组。
```

因为很多问题都可以转化为：

```text
按字典序排序后，相邻后缀之间的 LCP 信息。
```

一个重要性质：

```text
任意两个后缀 sa[l] 和 sa[r] 的 LCP
=
min(height[l+1], height[l+2], ..., height[r])
```

这个性质会把后缀之间的公共前缀问题转化成：

```text
height 数组上的区间最小值问题。
```

因此后缀数组经常搭配：

- RMQ；
- ST 表；
- 单调队列；
- 单调栈；
- 并查集；
- 二分答案；
- 滑动窗口。

## SA、rk、height 的含义

以：

```text
s = banana
```

为例，按字典序排序后：

```text
rank  suffix   start
1     a        6
2     ana      4
3     anana    2
4     banana   1
5     na       5
6     nana     3
```

所以：

```text
sa[1] = 6
sa[2] = 4
sa[3] = 2
sa[4] = 1
sa[5] = 5
sa[6] = 3
```

`rk[i]` 是反向数组：

```text
rk[6] = 1
rk[4] = 2
rk[2] = 3
rk[1] = 4
rk[5] = 5
rk[3] = 6
```

`height[i]` 表示相邻后缀的 LCP：

```text
height[1] = 0
height[2] = LCP(sa[2], sa[1]) = LCP(ana, a) = 1
height[3] = LCP(anana, ana) = 3
height[4] = LCP(banana, anana) = 0
height[5] = LCP(na, banana) = 0
height[6] = LCP(nana, na) = 2
```

## 倍增构造 SA 的基本思想

倍增法每一轮按长度为 `2k` 的前缀排序。

如果已经知道每个后缀前 `k` 个字符的排名，那么长度 `2k` 的比较可以转化为二元组比较：

```text
(rk[i], rk[i + k])
```

每轮：

```text
k = 1, 2, 4, 8, ...
```

直到所有后缀排名互不相同。

## height 数组怎么求？

设当前后缀起点为 `i`，它在 SA 中的排名是：

```text
rk[i]
```

它前一个后缀是：

```text
j = sa[rk[i] - 1]
```

`height[rk[i]]` 就是：

```text
LCP(s[i...], s[j...])
```

求 height 时有一个经典性质：

```text
相邻位置的 LCP 至少可以从上一次的 LCP - 1 开始尝试。
```

所以可以在线性时间内求出 height。

## 本阶段常见错误

- `sa` 和 `rk` 含义写反；
- `height[i]` 误认为是第 i 个后缀和原串的 LCP；
- 下标从 0 开始和从 1 开始混用；
- 倍增时第二关键字越界没有处理；
- 基数排序数组开小；
- 字符集没有离散化；
- P3809 数据较大，普通 `sort + substr` 会超时；
- height[1] 应为 0。

## 本质不同子串数量

一个长度为 `n` 的字符串，所有子串数量是：

```text
n * (n + 1) / 2
```

如果从后缀角度看：

```text
每个子串都是某个后缀的前缀。
```

排名第 `i` 的后缀 `sa[i]` 长度为：

```text
n - sa[i] + 1
```

它能贡献这么多个前缀。

但其中有一部分已经在前一个后缀中出现过，重复数量正好是：

```text
height[i]
```

所以第 `i` 个后缀新增的本质不同子串数量是：

```text
n - sa[i] + 1 - height[i]
```

总答案：

```text
sum(n - sa[i] + 1 - height[i])
```

等价于：

```text
n * (n + 1) / 2 - sum(height[i])
```

## 最长重复子串

如果某个子串至少出现两次，那么它一定是两个后缀的公共前缀。

所有相邻后缀的 LCP 都在 height 中。

所以：

```text
最长重复子串长度 = max(height[i])
```

如果要输出具体子串，可以记录最大 height 对应的 `sa[i]`。

## 出现至少 k 次的最长子串

如果一个长度为 `L` 的子串至少出现 `k` 次，那么在 SA 排序后，一定存在连续 `k` 个后缀，它们的公共前缀长度至少为 `L`。

对于连续 `k` 个后缀：

```text
sa[i], sa[i+1], ..., sa[i+k-1]
```

它们的公共前缀长度是：

```text
min(height[i+1], height[i+2], ..., height[i+k-1])
```

所以 P2852 可以转化为：

```text
在 height 数组中，找连续 k-1 个数的最小值的最大值。
```

可用：

- 二分答案；
- 单调队列；
- RMQ；
- 滑动窗口。

## 本阶段常见错误

- 本质不同子串忘记用 long long；
- 重复子串误写成 `max(height) + 1`；
- 出现至少 k 次时窗口长度应该是 `k - 1` 个 height；
- 连续 k 个后缀对应的 height 区间下标容易错；
- 如果字符串是数列，字符集要先离散化；
- 只会模板，不会把问题转成 height 上的区间最小值。

## 两个字符串最长公共子串

给定：

```text
A
B
```

拼接：

```text
T = A + 分隔符 + B
```

求出 T 的 SA 和 height。

然后遍历相邻后缀：

```text
如果 sa[i] 和 sa[i-1] 分别来自 A 和 B，
就用 height[i] 更新答案。
```

因为最长公共子串一定对应两个后缀的公共前缀。

## 多字符串最长公共子串

对于多个字符串：

```text
S1, S2, ..., Sk
```

拼接成：

```text
S1 + sep1 + S2 + sep2 + ... + Sk
```

要求分隔符：

```text
每个分隔符不同；
分隔符不能出现在原字符串中；
分隔符的字典序处理要稳定。
```

常见做法：

```text
1. 求 SA 和 height；
2. 二分答案 L；
3. 判断是否存在一段连续后缀，使其中包含所有字符串的后缀，且这一段内部的 height 都 >= L。
```

也可以用滑动窗口维护当前窗口覆盖了哪些字符串。

## LCP 查询

后缀 `i` 和后缀 `j` 的 LCP：

```text
设 x = rk[i], y = rk[j]
如果 x > y，交换 x 和 y
LCP(i, j) = min(height[x+1 ... y])
```

所以可以在 height 上建：

```text
ST 表 / 线段树 / RMQ
```

实现 O(1) 或 O(log n) 查询任意两个后缀的 LCP。

## 本阶段常见错误

- 拼接多个字符串时使用相同分隔符；
- 分隔符出现在原字符串中；
- 判断相邻后缀来源时把分隔符后缀也算进去；
- 两字符串公共子串只看相邻后缀，但忘记检查来源不同；
- 多字符串公共子串的窗口没有覆盖所有字符串；
- LCP 查询时 height 区间应为 `rk` 之间的开闭区间，容易错一位。

## P4248：height 的区间最小值贡献

任意两个后缀的 LCP 可以看成 height 数组上一段区间的最小值。

问题：

```text
求所有后缀两两 LCP 之和。
```

就可以转化为：

```text
求 height 数组所有区间最小值之和。
```

典型做法：

```text
用单调栈求每个 height[i] 作为区间最小值能覆盖多少个区间。
```

如果：

```text
height[i] 左边第一个小于它的位置是 L；
右边第一个小于等于它的位置是 R；
```

那么它作为最小值的贡献次数为：

```text
(i - L) * (R - i)
```

贡献为：

```text
height[i] * (i - L) * (R - i)
```

## P2178：height 从大到小合并

如果两个后缀的 LCP 至少为 `r`，说明它们在 height 数组上被若干个 `height >= r` 的边连接在一起。

可以把相邻后缀看作点之间的边：

```text
sa[i-1] -- sa[i]，边权为 height[i]
```

按边权从大到小加入这些边，并用并查集合并集合。

这样每个连通块中的任意两个后缀都至少有当前长度的公共前缀。

并查集可以维护：

- 集合大小；
- 最大权值；
- 次大权值；
- 最小权值；
- 次小权值；
- 当前能产生多少后缀对；
- 最大乘积贡献。

## 后缀数组与后缀自动机的对比

后缀数组更适合：

```text
离线处理固定字符串；
需要字典序；
需要后缀之间 LCP；
需要所有后缀排序；
需要和 RMQ / 单调栈结合。
```

后缀自动机更适合：

```text
在线加入字符；
统计不同子串；
统计子串出现次数；
多次匹配其他字符串；
动态性更强。
```

很多题 SA 和 SAM 都能做，但建模角度不同。

## 本阶段常见错误

- 所有后缀两两 LCP 没有转成 height 区间最小值；
- 单调栈左右边界条件没有一边严格、一边非严格，导致重复计数；
- 并查集合并时没有维护集合大小和极值；
- P2178 中权值乘积可能为负，最大值初始化错误；
- 多字符串题和并查集题中原始后缀位置与 SA 排名混淆；
- 只背模板，不会抽象出 height 的“边权”含义。

或者：

```text
本题需要把任意两个后缀的 LCP 转成 height 区间最小值。
```

## 2. `sa`、`rk`、`height` 在本题中的作用是什么？

例如：

```text
sa 用于按字典序枚举后缀。
rk 用于定位某个后缀的排名。
height 用于表示相邻后缀的 LCP。
```

## 3. 是否需要 LCP 查询？

如果需要任意两个后缀的 LCP，要写出：

```text
LCP(i, j) = min(height[rk[i]+1 ... rk[j]])
```

并考虑是否需要 ST 表或线段树。

## 4. 是否是多字符串问题？

如果是，要明确：

```text
每个后缀属于哪个原字符串；
分隔符是否唯一；
窗口是否覆盖了所有字符串。
```

## 5. 是否需要把 height 当成数组来处理？

常见处理方式：

```text
max(height)
height 滑动窗口最小值
height 区间 RMQ
height 所有区间最小值贡献
height 从大到小并查集合并
```

## 6. 是否能用 SAM 替代？

如果题目更偏：

```text
不同子串数量
出现次数
在线扩展
多次匹配
```

可以比较后缀自动机是否更自然。

# 后缀数组与其他字符串算法的选择

## 更适合后缀数组的情况

- 需要所有后缀的字典序；
- 需要比较任意两个后缀的 LCP；
- 需要统计所有后缀两两贡献；
- 需要处理多字符串公共子串；
- 需要固定字符串的离线分析；
- 需要与 RMQ、单调栈、并查集结合。

## 更适合字符串哈希的情况

- 只需要快速比较子串是否相等；
- 可以接受极低概率碰撞；
- 题目可以用二分 + 判重解决；
- 不需要完整后缀排序。

## 更适合 KMP / Z 函数的情况

- 单模式串匹配；
- 只关心前缀和后缀；
- 不需要任意后缀之间的 LCP；
- 不需要所有后缀排序。

## 更适合 Manacher 的情况

- 核心是回文子串；
- 需要每个中心的回文半径；
- 后缀排序不是主要需求。

## 更适合后缀自动机 SAM 的情况

- 统计不同子串；
- 统计子串出现次数；
- 在线加入字符；
- 多字符串匹配；
- 需要 parent tree 或 endpos 集合。

# 四类后缀数组问题的区别

## 1. 模板排序类

目标：

```text
构造 sa、rk、height。
```

代表题：

```text
P3809
```

## 2. height 基础应用类

目标：

```text
用 height 统计不同子串、最长重复子串、出现至少 k 次的子串。
```

代表题：

```text
P2408
LeetCode 1044
P2852
SPOJ DISUBSTR
```

## 3. 多字符串公共子串类

目标：

```text
拼接多个字符串，利用 height 和后缀来源求公共子串。
```

代表题：

```text
SPOJ LCS
POJ2774
P5546
P2463
```

## 4. height 贡献综合类

目标：

```text
把 height 看成数组或边权，用单调栈、并查集统计复杂贡献。
```

代表题：

```text
P4248
P2178
P3181
P1117
```

# 最终掌握标准

学完后缀数组后，看到一道题应该能快速判断：

1. 是否需要所有后缀的字典序？
2. 是否需要 `sa`、`rk`、`height` 三个数组？
3. 本题真正用的是后缀排序，还是 height 数组？
4. 是否能把子串问题转成“后缀的前缀”问题？
5. 是否需要统计本质不同子串？
6. 是否需要求最长重复子串或出现至少 k 次的子串？
7. 是否是多个字符串的公共子串问题？
8. 是否需要用 RMQ 查询任意两个后缀 LCP？
9. 是否能把所有后缀两两 LCP 转成 height 区间最小值贡献？
10. 是否更适合使用字符串哈希或后缀自动机？

如果这些问题都能回答出来，说明你已经不只是会背后缀数组模板，而是能真正用 SA 和 height 数组建模。
