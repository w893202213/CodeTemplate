# 一、基础模板题：单点修改 + 区间查询

## 学习目标

这一阶段只需要掌握最基础的树状数组模型：

- `lowbit(x) = x & -x`
- 单点加：`add(pos, delta)`
- 前缀和查询：`sum(pos)`
- 区间和查询：`sum(r) - sum(l - 1)`
- 下标从 `1` 开始维护

这一阶段不要急着学习二维树状数组或复杂离线题，先把最基础模板写熟。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P3374 【模板】树状数组 1 | https://www.luogu.com.cn/problem/P3374 | 单点加 + 区间求和，最标准入门题 |
| 2 | 洛谷 P2068 统计和 | https://www.luogu.com.cn/problem/P2068 | 初始全 0，单点加 + 区间和查询 |
| 3 | 力扣 307 区域和检索 - 数组可修改 | https://leetcode.cn/problems/range-sum-query-mutable/ | 面试版树状数组，封装类设计 |
| 4 | 洛谷 P3374 重新手写一遍 | https://www.luogu.com.cn/problem/P3374 | 用于巩固模板，建议不看代码独立重写 |

## 本阶段核心模板

```cpp
int lowbit(int x) {
    return x & -x;
}

void add(int x, long long v) {
    for (int i = x; i <= n; i += lowbit(i)) {
        tr[i] += v;
    }
}

long long query(int x) {
    long long res = 0;
    for (int i = x; i > 0; i -= lowbit(i)) {
        res += tr[i];
    }
    return res;
}

long long rangeQuery(int l, int r) {
    return query(r) - query(l - 1);
}
```

## 本阶段要点

树状数组的本质不是“树”，而是用数组维护若干段前缀贡献。

你需要真正理解：

```text
tr[x] 维护的是以 x 结尾、长度为 lowbit(x) 的一段区间和。
```

例如：

```text
tr[8] 维护 [1, 8]
tr[7] 维护 [7, 7]
tr[6] 维护 [5, 6]
tr[4] 维护 [1, 4]
```

# 二、差分树状数组与权值树状数组入门

## 学习目标

这一阶段开始理解树状数组的两个经典变形：

1. **差分树状数组**
   - 支持区间修改、单点查询；
   - 本质是在差分数组上做单点加和前缀查询。

2. **权值树状数组**
   - 下标不再表示位置，而是表示数值大小；
   - 常用于统计“比当前数小/大”的个数；
   - 常与离散化一起出现。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P3368 【模板】树状数组 2 | https://www.luogu.com.cn/problem/P3368 | 区间加 + 单点查询，差分树状数组模板 |
| 2 | 洛谷 P5057 [CQOI2006] 简单题 | https://www.luogu.com.cn/problem/P5057 | 区间翻转 + 单点查询，练习差分思想与奇偶判断 |
| 3 | 洛谷 P1908 逆序对 | https://www.luogu.com.cn/problem/P1908 | 离散化 + 权值树状数组统计逆序对 |
| 4 | 洛谷 P1966 [NOIP2013 提高组] 火柴排队 | https://www.luogu.com.cn/problem/P1966 | 排序映射 + 逆序对，练建模转换 |
| 5 | 力扣 315 计算右侧小于当前元素的个数 | https://leetcode.cn/problems/count-of-smaller-numbers-after-self/ | 面试常见题，离散化 + 权值树状数组 |

## 1. 差分树状数组

如果原数组是：

```text
a[1], a[2], ..., a[n]
```

定义差分数组：

```text
d[i] = a[i] - a[i - 1]
```

那么：

```text
a[x] = d[1] + d[2] + ... + d[x]
```

所以如果要给区间 `[l, r]` 加上 `k`，只需要：

```cpp
add(l, k);
add(r + 1, -k);
```

查询 `a[x]` 时：

```cpp
query(x);
```

## 2. 权值树状数组

普通树状数组中：

```text
下标 = 原数组位置
```

权值树状数组中：

```text
下标 = 数值大小或离散化后的排名
tr[x] = 数值 x 出现了多少次
```

常用查询：

```text
query(x) = 当前已经出现的、值 <= x 的数的个数
```

因此可以解决：

- 逆序对
- 右侧小于当前元素的个数
- 三元上升子序列
- 动态排名
- 前驱后继的近似维护

## 本阶段常见错误

- 忘记离散化；
- 离散化后下标从 0 开始，导致 `lowbit(0)` 死循环；
- 统计逆序对时方向写反；
- 处理相等元素时没有想清楚是 `<`、`<=`、`>` 还是 `>=`；
- 差分树状数组中忘记对 `r + 1` 做反向修改。

## 1. HH 的项链：典型离线树状数组

问题本质：

```text
多次询问区间 [l, r] 中不同数字的个数。
```

做法：

1. 将询问按右端点 `r` 从小到大排序；
2. 从左到右扫描原数组；
3. 对于当前数字 `a[i]`：
   - 如果它之前出现过，就把上一次出现位置的贡献清掉；
   - 再把当前位置贡献设为 1；
4. 对于询问 `[l, r]`，答案就是：

```text
query(r) - query(l - 1)
```

这道题非常重要，因为它体现了树状数组解决离线区间问题的能力。

## 2. 三元上升子序列

目标是统计：

```text
i < j < k 且 a[i] < a[j] < a[k]
```

可以枚举中间点 `j`：

```text
答案 += 左侧比 a[j] 小的数量 * 右侧比 a[j] 大的数量
```

两个方向各扫一遍即可：

- 从左到右：统计 `leftSmall[j]`
- 从右到左：统计 `rightBig[j]`

## 3. MooFest

这类题的核心是把公式拆开。

当按照听力值排序后，当前牛与之前牛的贡献可以写成：

```text
v[i] * sum(|x[i] - x[j]|)
```

再把绝对值拆成：

```text
左侧数量 * x[i] - 左侧坐标和
+
右侧坐标和 - 右侧数量 * x[i]
```

因此通常需要两个树状数组：

```text
cntBIT：维护数量
sumBIT：维护坐标和
```

## 1. 二维树状数组基础模板

单点加：

```cpp
void add(int x, int y, int v) {
    for (int i = x; i <= n; i += lowbit(i)) {
        for (int j = y; j <= m; j += lowbit(j)) {
            tr[i][j] += v;
        }
    }
}
```

前缀查询：

```cpp
int query(int x, int y) {
    int res = 0;
    for (int i = x; i > 0; i -= lowbit(i)) {
        for (int j = y; j > 0; j -= lowbit(j)) {
            res += tr[i][j];
        }
    }
    return res;
}
```

子矩阵查询：

```cpp
sum(x1, y1, x2, y2)
= query(x2, y2)
- query(x1 - 1, y2)
- query(x2, y1 - 1)
+ query(x1 - 1, y1 - 1)
```

## 2. 二维区间加 + 区间求和

二维区间加、区间求和通常需要维护四个树状数组：

```text
t1, t2, t3, t4
```

思想类似一维中的“两个树状数组维护区间加区间和”。

如果你暂时觉得难，可以先跳过 P4514，等一维区间修改区间查询完全熟练后再回来做。

## 3. CDQ 分治 + 树状数组

三维偏序问题通常思路是：

1. 第一维排序；
2. 第二维用 CDQ 分治处理；
3. 第三维用树状数组维护计数。

洛谷 P3810 是这个方向的经典模板题。

这类题本质上不是“树状数组本身难”，而是“离线排序 + 分治 + 数据结构协同”难。

# 每道题的复盘模板

建议每做完一道树状数组题，都写下面几句话。

## 1. 当前树状数组的下标表示什么？

例如：

```text
本题树状数组下标表示原数组位置。
```

或者：

```text
本题树状数组下标表示离散化后的数值排名。
```

## 2. `tr[i]` 维护的是什么？

例如：

```text
tr[i] 维护一段位置区间的元素和。
```

或者：

```text
tr[i] 维护若干数值排名出现的次数。
```

## 3. 查询的是前缀和、区间和，还是排名数量？

例如：

```text
query(x) 表示前 x 个位置的元素和。
```

或者：

```text
query(x) 表示当前已经出现的、排名 <= x 的元素数量。
```

## 4. 是否需要离散化？

如果数值范围很大，例如：

```text
a[i] <= 10^9
```

但只关心大小关系，就通常需要离散化。

## 5. 是在线问题还是离线问题？

例如：

```text
HH 的项链需要将询问按右端点排序，是离线问题。
```

# 树状数组与线段树的选择

## 优先考虑树状数组的情况

- 单点修改 + 前缀/区间求和；
- 区间修改 + 单点查询；
- 区间修改 + 区间求和；
- 统计排名、逆序对、偏序数量；
- 需要代码短、常数小；
- 信息满足“前缀可减”的性质。

## 更适合线段树的情况

- 维护区间最大值、最小值并且有复杂修改；
- 需要区间覆盖、区间翻转、区间取 min/max；
- 需要维护最大子段和等复杂合并信息；
- lazy 标记种类多且不容易转成差分；
- 需要支持更复杂的区间信息合并。

一句话总结：

```text
树状数组适合“前缀可减”的问题；
线段树适合“区间信息可合并”的问题。
```

# 最终掌握标准

学完树状数组后，看到一道题应该能快速判断：

1. 是否能转成前缀和？
2. 是否能用差分处理区间修改？
3. 是否需要离散化？
4. 树状数组下标表示位置还是数值？
5. 查询的是前缀和、区间和、排名，还是出现次数？
6. 是否需要离线排序？
7. 是否需要二维树状数组或 CDQ 分治？

如果这些问题都能回答出来，说明你已经不只是会背模板，而是能用树状数组建模了。
