# 数据结构与算法：博弈论刷题学习路线

> 面向零基础、以刷题为导向。目标不是系统学习经济学博弈论，而是掌握算法题中常见的“双方最优、轮流行动、判断胜负或最优得分”的解题套路。

## 0. 先说清楚：刷题里的“博弈论”学什么

算法竞赛和面试题中的博弈论通常分两大类：

1. **公平组合游戏 / Nim / SG 函数**
   - 两个人轮流行动。
   - 双方可选操作集合相同。
   - 局面通常会终止。
   - 常问：“先手是否必胜？”
   - 关键词：必胜态、必败态、Nim 异或和、mex、SG 函数、打表找规律。

2. **博弈型动态规划**
   - 两个人轮流从数组、石子、区间中选择。
   - 双方都最优。
   - 常问：“先手最多得多少分？”“先手能否赢？”
   - 关键词：状态定义、当前玩家视角、分数差 DP、区间 DP、记忆化搜索。

建议学习顺序：**先学状态胜负判断，再学 Nim/SG，再学博弈 DP**。这样最贴近刷题的上手路径。

## 2. 核心思想：必胜态与必败态

这是所有刷题博弈论的入口。

### 2.1 定义

- **必败态**：轮到当前玩家行动时，只要双方都最优，当前玩家必输。
- **必胜态**：轮到当前玩家行动时，存在一种走法可以让对手进入必败态。

### 2.2 转移原则

记住这两句话：

1. **如果当前状态能走到至少一个必败态，那么当前状态是必胜态。**
2. **如果当前状态所有后继都是必胜态，那么当前状态是必败态。**

### 2.3 最小模板

```cpp
bool win(state s) {
    if (s is terminal) return false;
    for (auto next : moves(s)) {
        if (!win(next)) return true;
    }
    return false;
}
```

加上记忆化：

```cpp
unordered_map<State, bool> memo;

bool win(State s) {
    if (terminal(s)) return false;
    if (memo.count(s)) return memo[s];

    for (State nxt : nextStates(s)) {
        if (!win(nxt)) return memo[s] = true;
    }
    return memo[s] = false;
}
```

## 3. 第一阶段：取石子类入门

目标：建立“当前状态能否转移到必败态”的直觉。

### 3.1 必做题

| 顺序 | 题目 | 重点 |
|---|---|---|
| 1 | LeetCode 292. Nim Game | 最经典的取石子入门题 |
| 2 | LeetCode 1025. Divisor Game | 状态胜负 DP |
| 3 | HDU / POJ 常见 Bash Game | 每次取 1 到 m 个，找模数规律 |

### 3.2 你要掌握的结论

#### Bash Game

有 `n` 个石子，每次可以取 `1..m` 个，取到最后一个的人赢。

结论：

```text
n % (m + 1) == 0 -> 先手必败
否则 -> 先手必胜
```

推导方式：

- `0` 是必败态。
- `1..m` 都能一步拿完，是必胜态。
- `m + 1` 无论拿多少，都会给对手留下 `0..m-1` 中的必胜态，所以是必败态。
- 后面循环。

### 3.3 刷题要求

每做一道题都写下：

- 状态是什么？
- 终止态是什么？
- 当前玩家有哪些行动？
- 是否能走到对手必败态？
- 有没有周期规律？

## 4. 第二阶段：Nim 游戏与异或

目标：掌握多堆石子的核心结论。

### 4.1 标准 Nim

有若干堆石子，每次选择一堆，拿走任意正数个石子，拿到最后一个的人赢。

结论：

```text
所有堆数量的异或和 xorSum == 0 -> 先手必败
所有堆数量的异或和 xorSum != 0 -> 先手必胜
```

示例：

```text
[3, 4, 5]
3 ^ 4 ^ 5 = 2
异或和不为 0，先手必胜
```

### 4.2 为什么异或有用

你不用一开始完全证明，但要理解两个事实：

1. 如果当前异或和是 `0`，无论你怎么拿，都会让异或和变成非 `0`。
2. 如果当前异或和非 `0`，一定能通过改变某一堆，让异或和变成 `0`。

刷题时，这两个事实足够用。

### 4.3 必做题

| 顺序 | 题目 | 重点 |
|---|---|---|
| 1 | 标准 Nim 模板题 | 异或和判胜负 |
| 2 | Misere Nim / 反常 Nim | “拿最后一个输”的特殊处理 |
| 3 | LeetCode 810. Chalkboard XOR Game | 异或与奇偶性 |

### 4.4 代码模板

```cpp
bool firstWin(vector<int>& piles) {
    int x = 0;
    for (int v : piles) x ^= v;
    return x != 0;
}
```

## 5. 第三阶段：SG 函数与 mex

目标：能处理“不是标准 Nim，但可以拆成若干独立子游戏”的题。

### 5.1 SG 函数是什么

对一个状态 `s`，定义：

```text
SG(s) = mex({ SG(t) | t 是 s 的所有后继状态 })
```

终止态：

```text
SG(terminal) = 0
```

判断胜负：

```text
SG(s) == 0 -> 当前玩家必败
SG(s) != 0 -> 当前玩家必胜
```

### 5.2 多个独立游戏如何合并

如果一个局面可以拆成多个独立子游戏：

```text
总 SG = SG(子游戏1) ^ SG(子游戏2) ^ ... ^ SG(子游戏k)
```

判断：

```text
总 SG == 0 -> 先手必败
总 SG != 0 -> 先手必胜
```

### 5.3 SG 模板

```cpp
int sg(int x) {
    if (x == 0) return 0;
    if (memo[x] != -1) return memo[x];

    unordered_set<int> seen;
    for (int nxt : nextStates(x)) {
        seen.insert(sg(nxt));
    }

    int g = 0;
    while (seen.count(g)) g++;
    return memo[x] = g;
}
```

### 5.4 学习步骤

1. 先用暴力 SG 计算小范围结果。
2. 打印 `SG(0), SG(1), ..., SG(n)`。
3. 观察周期、分段、公式。
4. 证明或直接在题目范围内使用规律。

### 5.5 必做题型

| 题型 | 典型描述 | 方法 |
|---|---|---|
| 取石子集合 | 每次可取集合 `S` 中的数量 | SG / 周期 |
| 拆分游戏 | 一堆可拆成两堆 | SG + mex |
| 多个独立局面 | 多堆各自变化 | 子 SG 异或 |
| DAG 上移动棋子 | 棋子沿有向边移动 | 图上 SG |
| 翻硬币 / 翻硬币变体 | 一次操作影响局部 | 找子游戏或打表 |

## 6. 第四阶段：博弈型动态规划

目标：处理 LeetCode 上最常见的“Stone Game”系列。

### 6.1 什么时候用 DP，而不是 Nim/SG

满足这些特征时，优先想 DP：

- 双方拿分数，不只是判断能不能走。
- 操作会改变区间，比如从左端或右端取。
- 双方行动规则相同，但收益不同。
- 问“先手最终得分是否更多”。

### 6.2 最重要的状态定义：分数差

很多博弈 DP 都可以定义：

```text
dp[l][r] = 当前玩家在区间 [l, r] 内，最多能比对手多拿多少分
```

转移：

```text
选左端：piles[l] - dp[l + 1][r]
选右端：piles[r] - dp[l][r - 1]

dp[l][r] = max(选左端, 选右端)
```

为什么要减？

因为你拿完以后，轮到对手在剩余区间成为“当前玩家”。`dp` 表示的是当前玩家相对对手的优势，所以要反过来减掉对手的最优优势。

### 6.3 经典模板

```cpp
bool predictTheWinner(vector<int>& nums) {
    int n = nums.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));

    for (int i = 0; i < n; i++) dp[i][i] = nums[i];

    for (int len = 2; len <= n; len++) {
        for (int l = 0; l + len - 1 < n; l++) {
            int r = l + len - 1;
            dp[l][r] = max(nums[l] - dp[l + 1][r],
                           nums[r] - dp[l][r - 1]);
        }
    }

    return dp[0][n - 1] >= 0;
}
```

## 9. 解题套路速查

### 9.1 看到“先手是否必赢”

优先问：

1. 状态是否只有一个数字？
2. 是否可以从小到大 DP？
3. 是否有取石子模数规律？
4. 是否是 Nim 异或？
5. 是否需要 SG？

### 9.2 看到“双方都最优，最多得多少分”

优先问：

1. 是否能用当前玩家分数差？
2. 是否是区间 `[l, r]`？
3. 当前操作后是否轮到对手？
4. 转移是否是 `gain - dp[next]`？

### 9.3 看到“多个独立部分”

优先问：

1. 每次操作是否只影响一个部分？
2. 每个部分是否可以单独计算 SG？
3. 总局面是否是各部分 SG 的异或？

### 9.4 看到“规则很奇怪”

优先做：

1. 暴力搜索小范围。
2. 打表。
3. 看周期、奇偶、模数。
4. 再尝试证明。

## 10. 常见坑

### 坑 1：把“我能赢”写成“我能拿到更多”

有些题只问胜负，有些题问分数。先看清目标。

### 坑 2：忘记双方都最优

博弈题不是模拟随机行为。每一步都假设对手会选择对你最不利的策略。

### 坑 3：SG 只能直接用于公平组合游戏

如果两名玩家可用操作不同，或收益不同，不能直接套 SG。

### 坑 4：区间 DP 的视角混乱

推荐统一使用：

```text
dp[l][r] = 当前玩家最多领先对手多少
```

这样转移最清楚。

### 坑 5：打表只看前几项就下结论

找周期时至少多打印一些，比如 `0..100` 或 `0..1000`，再考虑证明。

## 13. 最小代码模板合集

### 13.1 胜负 DP

```cpp
vector<int> memo;

bool win(int x) {
    if (x == 0) return false;
    if (memo[x] != -1) return memo[x];

    for (int nxt : nextStates(x)) {
        if (!win(nxt)) return memo[x] = true;
    }
    return memo[x] = false;
}
```

### 13.2 SG + mex

```cpp
vector<int> memo;

int sg(int x) {
    if (x == 0) return 0;
    if (memo[x] != -1) return memo[x];

    unordered_set<int> seen;
    for (int nxt : nextStates(x)) {
        seen.insert(sg(nxt));
    }

    int g = 0;
    while (seen.count(g)) g++;
    return memo[x] = g;
}
```

### 13.3 多子游戏 SG 异或

```cpp
bool firstWin(vector<int>& states) {
    int x = 0;
    for (int s : states) {
        x ^= sg(s);
    }
    return x != 0;
}
```

### 13.4 分数差区间 DP

```cpp
int scoreDiff(vector<int>& a) {
    int n = a.size();
    vector<vector<int>> dp(n, vector<int>(n));

    for (int i = 0; i < n; i++) dp[i][i] = a[i];

    for (int len = 2; len <= n; len++) {
        for (int l = 0; l + len - 1 < n; l++) {
            int r = l + len - 1;
            dp[l][r] = max(a[l] - dp[l + 1][r],
                           a[r] - dp[l][r - 1]);
        }
    }

    return dp[0][n - 1];
}
```

## 14. 最后建议

博弈论刷题不要一开始追求证明完整性。更高效的路径是：

1. 先会判断题型。
2. 再会写暴力 / DP / SG。
3. 然后打表找规律。
4. 最后补证明。

刷到 20 道左右，你会发现大多数题都在问同一件事：

```text
我能不能把对手送到一个他不想面对的状态？
```

这个问题，就是算法博弈论的核心。
