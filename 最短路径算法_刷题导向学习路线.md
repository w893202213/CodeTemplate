# 最短路径问题到底解决什么？

最短路径问题通常研究：

```text
在一个带权图中，从某个起点到其他点，或者任意两点之间，路径权值和最小是多少。
```

常见类型：

- 单源最短路：从一个起点到所有点；
- 单点到单点最短路：从起点到终点；
- 多源最短路：多个起点到所有点；
- 全源最短路：任意两点之间；
- 次短路 / 第 k 短路；
- 最短路计数；
- 分层图最短路；
- 网格图最短路；
- 带限制条件的最短路；
- 最短路 + DP；
- 最短路 + 二分答案。

本路线重点学习：

```text
Dijkstra 算法及其变形。
```

# Dijkstra 的适用条件

Dijkstra 用于：

```text
边权非负的单源最短路径问题。
```

它不能直接处理负权边。

如果存在负权边，应考虑：

- Bellman-Ford；
- SPFA；
- Johnson；
- 拓扑图最短路；
- 重新建模避免负权。

一句话记忆：

```text
没有负边，用 Dijkstra。
有负边，先别急着用 Dijkstra。
```

## 朴素 Dijkstra 模板

适合：

```text
n 较小，m 较大，图比较稠密。
```

核心复杂度：

```text
O(n^2)
```

代码模板：

```cpp
const int N = 510;
const int INF = 0x3f3f3f3f;

int n, m;
int g[N][N];
int dist[N];
bool vis[N];

int dijkstra(int s, int t) {
    memset(dist, 0x3f, sizeof dist);
    memset(vis, false, sizeof vis);

    dist[s] = 0;

    for (int i = 1; i <= n; i++) {
        int u = -1;

        for (int j = 1; j <= n; j++) {
            if (!vis[j] && (u == -1 || dist[j] < dist[u])) {
                u = j;
            }
        }

        if (u == -1 || dist[u] == INF) break;

        vis[u] = true;

        for (int v = 1; v <= n; v++) {
            if (dist[v] > dist[u] + g[u][v]) {
                dist[v] = dist[u] + g[u][v];
            }
        }
    }

    return dist[t];
}
```

## 松弛操作是什么？

如果已经知道从起点到 `u` 的最短距离是：

```text
dist[u]
```

并且有一条边：

```text
u -> v，边权为 w
```

那么可以尝试用：

```text
dist[u] + w
```

更新 `dist[v]`：

```cpp
if (dist[v] > dist[u] + w) {
    dist[v] = dist[u] + w;
}
```

这一步叫：

```text
松弛操作。
```

## 本阶段常见错误

- 图中有重边时没有取最小边权；
- 初始化 `dist` 时不够大；
- `dist[u] + w` 超过 int，应该用 `long long`；
- 不可达点没有特殊处理；
- 把 Dijkstra 用在负权图上；
- 邻接矩阵默认值没有设为 INF；
- 起点距离没有设为 0；
- 无向图忘记加双向边。

# 二、标准核心：堆优化 Dijkstra

## 学习目标

这一阶段掌握最常用的 Dijkstra 写法：

```text
邻接表 + 优先队列小根堆
```

需要掌握：

- 邻接表建图；
- `priority_queue` 小根堆；
- 堆中可能存在过期状态；
- 用 `if (d != dist[u]) continue;` 或 `vis[u]` 跳过旧状态；
- 时间复杂度 `O(m log m)` 或 `O(m log n)`；
- 为什么不能简单删除堆中旧值。

| 6 | 力扣 1631 最小体力消耗路径 | https://leetcode.cn/problems/path-with-minimum-effort-path/ | 网格图 Dijkstra，路径代价为边权最大值 |

## 堆优化 Dijkstra 模板

适合：

```text
n 和 m 较大，图比较稀疏。
```

代码模板：

```cpp
using ll = long long;
using PII = pair<ll, int>;

const ll INF = 4e18;
const int N = 100000 + 10;

struct Edge {
    int to;
    ll w;
};

vector<Edge> g[N];
ll dist[N];
bool vis[N];

void dijkstra(int s) {
    priority_queue<PII, vector<PII>, greater<PII>> pq;

    fill(dist, dist + N, INF);
    fill(vis, vis + N, false);

    dist[s] = 0;
    pq.push({0, s});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (vis[u]) continue;
        vis[u] = true;

        for (auto e : g[u]) {
            int v = e.to;
            ll w = e.w;

            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}
```

也可以不用 `vis`，改成：

```cpp
if (d != dist[u]) continue;
```

这表示堆顶状态已经过期，就跳过。

## 为什么堆中会有过期状态？

假设某个点 `v` 的距离先被更新为：

```text
100
```

后来又被更新为：

```text
20
```

堆里会同时存在：

```text
(100, v)
(20, v)
```

当 `(20, v)` 被处理后，`dist[v] = 20`。  
之后再弹出 `(100, v)` 时，它已经不是当前最短距离，应该跳过。

## 本阶段常见错误

- `priority_queue` 默认是大根堆，忘记改成小根堆；
- pair 顺序写成 `{点编号, 距离}`，导致排序错；
- 没有跳过堆中过期状态；
- 使用 int 存距离，边权和溢出；
- P4779 中数据较大，朴素 Dijkstra 会超时；
- 有向图和无向图建边搞混；
- 起点到自身距离没有输出 0；
- 不可达点没有按题意输出。

## Dijkstra + 最短路计数

如果求从起点到每个点的最短路条数，可以在松弛时维护：

```text
cnt[v] = 到达 v 的最短路条数。
```

更新规则：

```cpp
if (dist[v] > dist[u] + w) {
    dist[v] = dist[u] + w;
    cnt[v] = cnt[u];
} else if (dist[v] == dist[u] + w) {
    cnt[v] += cnt[u];
}
```

注意：

```text
如果题目要求取模，必须每次加法取模。
```

## 正图与反图

P1629 这类题要求：

```text
从 1 到每个点的距离
+
每个点回到 1 的距离
```

第一部分可以在原图上跑 Dijkstra。

第二部分可以在反图上从 1 再跑一次 Dijkstra。

原因：

```text
原图中 i -> 1 的最短路
等价于
反图中 1 -> i 的最短路。
```

## 多源 Dijkstra

如果有多个源点，可以把所有源点的距离初始化为 0，并全部入堆：

```cpp
for (int s : sources) {
    dist[s] = 0;
    pq.push({0, s});
}
```

这样求的是：

```text
每个点到最近源点的最短距离。
```

## 变形权值

Dijkstra 不一定只能处理“路径边权和”。

只要满足某种“先弹出的状态已经最优”的性质，也可以变形。

例如力扣 1514：

```text
路径概率 = 边概率连乘
目标 = 最大化概率
```

可以用最大堆，每次取当前概率最大的点。

力扣 1631：

```text
路径代价 = 路径上最大边权
目标 = 最小化这个最大值
```

松弛时：

```cpp
newCost = max(dist[u], edgeCost);
```

## 本阶段常见错误

- 计数题中距离变小时没有把 `cnt[v]` 重置为 `cnt[u]`；
- 距离相等时没有累加方案数；
- 正反图题中反图建边方向写错；
- 多源 Dijkstra 忘记把所有源点入堆；
- 最大概率路径仍用最小堆；
- 路径最大边权题仍按加法松弛；
- 二分 + Dijkstra 中判定函数没有清空图或 dist。

## 次短路

次短路通常维护两个数组：

```text
dist1[v]：从起点到 v 的最短路
dist2[v]：从起点到 v 的严格次短路
```

松弛时可能出现三种情况：

```text
新距离 < dist1[v]
dist1[v] < 新距离 < dist2[v]
新距离 == dist1[v]，视题意决定是否计入
```

严格次短路要求：

```text
dist2[v] > dist1[v]
```

P2865 是非常经典的次短路题。

## 分层图最短路

如果题目中有：

```text
最多使用 k 次特殊操作
最多免费 k 条边
最多翻转 k 次状态
最多使用 k 张优惠券
```

就可以考虑分层图。

设状态：

```text
(node, used)
```

其中：

```text
node：当前点
used：已经使用了多少次特殊操作
```

普通边：

```text
(node, used) -> (next, used)，权值为 w
```

使用一次特殊操作：

```text
(node, used) -> (next, used + 1)，权值为 0 或特殊权值
```

最后答案通常是：

```text
min(dist[t][0], dist[t][1], ..., dist[t][k])
```

## Johnson 全源最短路

Johnson 用于：

```text
有负权边但无负环的全源最短路。
```

核心步骤：

1. 加一个超级源点；
2. 用 Bellman-Ford 求势能函数 `h[i]`；
3. 对每条边重新赋权：

```text
w'(u, v) = w(u, v) + h[u] - h[v]
```

4. 重新赋权后所有边非负；
5. 从每个点跑 Dijkstra；
6. 把答案转换回原权值：

```text
dist(u, v) = dist'(u, v) - h[u] + h[v]
```

Johnson 的关键不是“只会跑 Dijkstra”，而是：

```text
通过重标号把含负边图转成非负边图。
```

## 状态图最短路

很多题表面不是图，但可以把状态看作点。

例如：

```text
当前位置 + 已经拿到的钥匙集合
```

就是一个状态：

```text
(x, y, mask)
```

状态之间的转移就是边。

如果边权都是 1，通常用 BFS。  
如果边权不等，考虑 Dijkstra。

## 本阶段常见错误

- 次短路中没有维护严格大于最短路；
- 次短路更新顺序写错；
- 分层图边数扩大后数组开小；
- `dist[node][used]` 下标含义混乱；
- 免费边建错方向或层数；
- Johnson 中重新赋权公式写反；
- Johnson 最终答案没有还原；
- 有负环时仍继续跑 Dijkstra；
- 状态图中没有把所有状态都当成节点处理。

# 每道题的复盘模板

建议每做完一道最短路题，都写下面几个问题。

## 1. 图是如何建出来的？

例如：

```text
本题直接给出有向边。
```

或者：

```text
本题是网格图，每个格子是一个点，相邻格子之间有边。
```

或者：

```text
本题需要分层建图，状态为 (点编号, 已使用免费次数)。
```

## 2. 边权是否非负？

如果边权非负，可以考虑 Dijkstra。

如果有负权边，要考虑：

```text
Bellman-Ford
SPFA
Johnson
拓扑图最短路
```

## 3. 求的是哪种最短路？

例如：

```text
单源到所有点。
单源到单点。
多源到所有点。
全源最短路。
次短路。
带限制的最短路。
```

## 4. `dist` 的含义是什么？

普通题：

```text
dist[v] = 起点到 v 的最短距离。
```

分层图：

```text
dist[v][k] = 到达 v 且已经使用 k 次特殊操作的最短距离。
```

计数题：

```text
dist[v] = 最短距离；
cnt[v] = 最短路条数。
```

## 5. 松弛规则是什么？

普通 Dijkstra：

```text
dist[v] > dist[u] + w
```

最大概率路径：

```text
prob[v] < prob[u] * p
```

最小体力消耗路径：

```text
dist[v] > max(dist[u], edgeCost)
```

次短路：

```text
同时维护 dist1 和 dist2。
```

## 6. 是否需要反图、虚拟源点或分层图？

例如：

```text
往返最短路：正图一次，反图一次。
多源最短路：所有源点同时入堆。
有 k 次特殊操作：分层图。
Johnson：超级源点。
```

# Dijkstra 与其他最短路算法的选择

## 更适合 Dijkstra 的情况

- 边权非负；
- 单源最短路；
- 图较大；
- 稀疏图配合堆优化；
- 需要处理最短路计数、次短路、分层图等变形；
- 网格图边权非负。

## 更适合 BFS 的情况

- 所有边权都是 1；
- 无权图最短路；
- 状态图每次转移代价相同；
- 最短步数问题。

## 更适合 0-1 BFS 的情况

- 边权只有 0 和 1；
- 可以用双端队列优化；
- 比 Dijkstra 更快更简单。

## 更适合 Bellman-Ford / SPFA 的情况

- 存在负权边；
- 需要判断负环；
- 点边规模不太大；
- 题目明确适合松弛若干轮。

## 更适合 Floyd 的情况

- 点数较小；
- 需要任意两点间最短路；
- 通常 `n <= 500` 或更小；
- 实现简单但复杂度 `O(n^3)`。

## 更适合 Johnson 的情况

- 需要全源最短路；
- 图较稀疏；
- 可能存在负权边但无负环；
- Floyd 的 `O(n^3)` 过大。

# 最终掌握标准

学完 Dijkstra 为主的最短路径后，看到一道题应该能快速判断：

1. 图是否已经给出，还是需要自己建图？
2. 边权是否非负？
3. 是否适合 Dijkstra？
4. 图是稠密图还是稀疏图？
5. 该用朴素 Dijkstra，还是堆优化 Dijkstra？
6. 是否有重边、自环、不可达点？
7. 距离是否会爆 int？
8. 是否要求最短路条数？
9. 是否需要正反图、多源、虚拟源点？
10. 是否有免费次数、优惠券、状态限制，需要分层图？
11. 是否要求次短路或第 k 短路？
12. 是否存在负权边，需要 Johnson 或 Bellman-Ford？
13. 松弛规则是否还是普通加法，还是概率、最大边权、状态代价？

如果这些问题都能回答出来，说明你已经不是只会背 Dijkstra 模板，而是能用最短路思想完成建图、选算法和处理变形。
