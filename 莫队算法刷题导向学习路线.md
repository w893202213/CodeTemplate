# 莫队算法到底解决什么问题？

莫队算法主要解决一类**离线区间询问**问题。

典型形式：

```text
给定一个数组 a[1...n]，
有 m 个询问，每次询问区间 [l, r] 的某种统计量。
```

如果我们能在当前区间 `[L, R]` 的基础上，通过移动一个端点来快速更新答案：

```text
[L, R] -> [L - 1, R]
[L, R] -> [L + 1, R]
[L, R] -> [L, R - 1]
[L, R] -> [L, R + 1]
```

也就是能写出：

```cpp
add(pos);
del(pos);
```

那么就可以考虑莫队。

莫队的核心思想：

```text
把所有询问离线排序，让相邻询问的区间变化尽量小。
```

普通莫队通常复杂度为：

```text
O((n + m) sqrt(n))
```

# 莫队适用条件

## 适合莫队的情况

- 询问可以离线；
- 都是区间查询；
- 没有修改，或修改也可以离线处理；
- 当前答案可以通过加入/删除一个元素快速更新；
- 查询结果与询问顺序无关；
- 线段树、树状数组不好维护当前答案。

## 不适合莫队的情况

- 必须在线回答；
- 有复杂区间修改且不能离线；
- add/del 一个元素代价很高；
- 答案无法通过局部增删维护；
- 题目本身可以用更简单的前缀和、树状数组或线段树解决。

一句话判断：

```text
如果一个区间答案可以通过移动左右端点逐步维护，且询问能离线，就优先考虑莫队。
```

## 普通莫队基本模板

```cpp
struct Query {
    int l, r, id;
};

int n, m, block;
vector<Query> qs;

bool cmp(const Query& a, const Query& b) {
    int A = a.l / block;
    int B = b.l / block;

    if (A != B) return A < B;

    // 奇偶优化：减少右端点来回移动
    if (A & 1) return a.r > b.r;
    return a.r < b.r;
}

void add(int pos) {
    // 把 a[pos] 加入当前区间
}

void del(int pos) {
    // 把 a[pos] 从当前区间删除
}

void solve() {
    block = sqrt(n);
    sort(qs.begin(), qs.end(), cmp);

    int L = 1, R = 0;

    for (auto q : qs) {
        while (L > q.l) add(--L);
        while (R < q.r) add(++R);
        while (L < q.l) del(L++);
        while (R > q.r) del(R--);

        ans[q.id] = currentAnswer;
    }
}
```

## P2709 的 add / del 写法

题目答案：

```text
sum(cnt[x]^2)
```

加入一个数 `x`：

```cpp
void add(int pos) {
    int x = a[pos];
    cur -= 1LL * cnt[x] * cnt[x];
    cnt[x]++;
    cur += 1LL * cnt[x] * cnt[x];
}
```

删除一个数 `x`：

```cpp
void del(int pos) {
    int x = a[pos];
    cur -= 1LL * cnt[x] * cnt[x];
    cnt[x]--;
    cur += 1LL * cnt[x] * cnt[x];
}
```

## 本阶段必须理解

### 1. 为什么莫队要离线？

因为莫队需要把所有询问重新排序。

如果题目要求按输入顺序实时回答，就不能直接用普通莫队。

### 2. 为什么排序后复杂度能降低？

普通暴力处理每个询问可能每次都重新扫描区间。

莫队排序后，相邻询问的左右端点变化较小，所以可以复用上一个询问的答案。

### 3. 为什么要保存 `id`？

因为询问排序后顺序会改变，最终输出仍要按照原输入顺序输出。

## 本阶段常见错误

- 忘记保存原询问编号；
- `L`、`R` 初始值写错；
- `add` 和 `del` 不是互逆操作；
- 当前区间左右端点移动顺序混乱；
- 块长为 0，尤其当 `n` 很小时；
- 计数数组没有离散化导致越界；
- 答案可能超过 int，忘记使用 `long long`。

## 小 Z 的袜子：组合计数

题目要求：

```text
从区间 [L, R] 中随机选两只袜子，颜色相同的概率。
```

如果某种颜色出现了 `cnt[x]` 次，那么能组成的同色袜子对数是：

```text
C(cnt[x], 2)
```

总同色对数为：

```text
sum C(cnt[x], 2)
```

区间总方案数为：

```text
C(len, 2)
```

所以答案是：

```text
sum C(cnt[x], 2) / C(len, 2)
```

加入一个颜色 `x` 时：

```cpp
cur += cnt[x];
cnt[x]++;
```

因为新加入的一只袜子能和原来的 `cnt[x]` 只同色袜子配对。

删除一个颜色 `x` 时：

```cpp
cnt[x]--;
cur -= cnt[x];
```

## 异或序列：前缀异或转化

区间 `[l, r]` 的异或值为：

```text
pre[r] ^ pre[l - 1]
```

若要求异或值为 `k`，则：

```text
pre[r] ^ pre[l - 1] = k
```

转化为：

```text
pre[l - 1] = pre[r] ^ k
```

所以可以在莫队维护的前缀异或区间中统计满足条件的数对。

注意原数组询问 `[l, r]` 通常会转成前缀数组询问：

```text
[l - 1, r]
```

## 作业类题：莫队 + 值域结构

有些莫队题不仅要维护区间，还要在值域上查询：

```text
区间内值在 [a, b] 的元素个数
区间内值在 [a, b] 的不同元素个数
```

这时可以在莫队外再套：

- 值域分块；
- 树状数组；
- 线段树；
- 计数数组 + 块内统计。

## 本阶段常见错误

- 分数题没有约分；
- 区间长度为 1 时分母为 0，需要特判；
- 组合计数更新顺序写反；
- 前缀异或题中忘记把询问 `[l, r]` 转成 `[l - 1, r]`；
- 统计数对时重复计算或漏算；
- 值域过大时没有离散化。

# 三、带修改莫队：加入时间维度

## 学习目标

这一阶段学习带修改莫队，也叫三维莫队。

普通莫队只有两个维度：

```text
L, R
```

带修改莫队多了一个维度：

```text
L, R, T
```

其中：

```text
T = 当前已经执行到第几次修改
```

需要掌握：

- 把操作拆成查询和修改；
- 每个查询记录它之前发生了多少次修改；
- 支持应用修改；
- 支持撤销修改；
- 修改点在当前区间内时，需要同步更新答案。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P1903【模板】带修莫队 / 数颜色 | https://www.luogu.com.cn/problem/P1903 | 带修改莫队模板，区间不同颜色数 |
| 2 | 洛谷 P3688 [ZJOI2017] 树状数组 | https://www.luogu.com.cn/problem/P3688 | 可作为更复杂离线思想扩展，难度较高 |
| 3 | 洛谷 P2486 [SDOI2011] 染色 | https://www.luogu.com.cn/problem/P2486 | 不是莫队主解，可用于对比：路径修改更适合树剖 |
| 4 | Codeforces 940F Machine Learning | https://codeforces.com/problemset/problem/940/F | 带修莫队经典题，英文题，求区间 mex of frequency |

## 带修莫队询问结构

```cpp
struct Query {
    int l, r, t, id;
};
```

其中：

```text
l, r：询问区间
t：该询问之前已经发生的修改次数
id：原询问编号
```

修改结构：

```cpp
struct Change {
    int pos;
    int oldVal;
    int newVal;
};
```

## 应用修改与撤销修改

如果当前时间 `curT` 小于询问时间 `q.t`，就应用下一次修改：

```cpp
apply(++curT);
```

如果当前时间 `curT` 大于询问时间 `q.t`，就撤销当前修改：

```cpp
undo(curT--);
```

应用修改时要注意：

```text
如果修改的位置 pos 在当前区间 [L, R] 内，
需要先删除旧值贡献，再加入新值贡献。
```

伪代码：

```cpp
void apply(int t) {
    int p = changes[t].pos;

    if (L <= p && p <= R) {
        delValue(a[p]);
        addValue(changes[t].newVal);
    }

    a[p] = changes[t].newVal;
}
```

撤销修改：

```cpp
void undo(int t) {
    int p = changes[t].pos;

    if (L <= p && p <= R) {
        delValue(a[p]);
        addValue(changes[t].oldVal);
    }

    a[p] = changes[t].oldVal;
}
```

## 排序方式

带修莫队一般按三维排序：

```text
1. 左端点所在块
2. 右端点所在块
3. 时间
```

块长常见取法：

```text
n^(2/3)
```

也有题目直接取：

```text
sqrt(n)
```

但带修莫队中，`n^(2/3)` 通常更常见。

## 本阶段常见错误

- 查询没有记录当时的修改次数；
- 修改没有保存旧值，导致无法撤销；
- 修改点在当前区间内时没有更新答案；
- 修改点不在当前区间内却错误更新答案；
- 应用和撤销操作不互逆；
- 排序块长不合理导致超时；
- 颜色值很大时没有离散化。

## 树上莫队

树上莫队用于处理：

```text
树上路径询问
```

典型做法：

```text
1. 对树做欧拉序；
2. 把树上路径转化成序列上的区间；
3. 用莫队维护当前区间；
4. 一个点在欧拉序中可能出现两次，用 vis[x] 表示当前是否计入答案；
5. 如果 LCA 不在区间中，需要临时加入 LCA。
```

关键点：

```text
序列中出现两次的点，贡献会抵消。
```

所以 add/del 经常写成同一个函数：

```cpp
void toggle(int x) {
    if (vis[x]) remove(x);
    else add(x);
    vis[x] ^= 1;
}
```

## 回滚莫队

普通莫队要求：

```text
add 和 del 都容易写。
```

但有些题：

```text
add 容易，del 很难。
```

这时可以考虑回滚莫队。

回滚莫队的思想：

```text
按块处理左端点；
右端点只向右扩展；
每个询问的左侧零散部分临时加入；
算完后回滚临时影响。
```

适合：

- 删除困难；
- 加入容易；
- 答案可以保存和恢复；
- 一侧指针只增不减。

## 二次离线莫队

二次离线莫队用于进一步优化某些普通莫队会超时的问题。

大致思想：

```text
普通莫队移动指针时产生很多贡献查询；
把这些贡献查询再次离线处理。
```

这个方向比较难，建议在普通莫队、带修莫队、树上莫队都熟练后再学。

## 本阶段常见错误

- 欧拉序区间转化错误；
- LCA 没有临时计入答案；
- 同一个点出现两次时没有用 `vis` 切换贡献；
- 带修树上莫队中时间维和树上路径维护混乱；
- 回滚莫队中没有恢复临时加入的贡献；
- 二次离线莫队中贡献拆分方向错误。

# 每道题的复盘模板

建议每做完一道莫队题，都写下面几个问题。

## 1. 为什么这题能用莫队？

例如：

```text
本题是离线区间询问，答案可以通过移动左右端点并增删一个元素维护，因此可以用莫队。
```

## 2. `add(pos)` 做什么？

例如：

```text
把 a[pos] 加入当前区间，并更新 cnt 和当前答案。
```

## 3. `del(pos)` 做什么？

例如：

```text
把 a[pos] 从当前区间删除，并撤销它对当前答案的贡献。
```

## 4. 当前答案维护的是什么？

例如：

```text
currentAnswer = sum(cnt[x]^2)
```

或者：

```text
currentAnswer = 当前区间不同颜色数量
```

或者：

```text
currentAnswer = 当前区间满足异或为 k 的子区间数量
```

## 5. 是否需要离散化？

如果值域很大，例如：

```text
a[i] <= 10^9
```

而你需要用 `cnt[a[i]]`，就必须离散化。

## 6. 是否有修改？

如果有修改，需要写出：

```text
查询发生在第几次修改之后。
```

并分别实现：

```text
apply(change)
undo(change)
```

## 7. 是否在树上？

如果在树上，需要说明：

```text
如何用欧拉序把路径转成区间；
LCA 是否需要额外加入。
```

# 莫队与其他算法的选择

## 更适合莫队的情况

- 离线区间询问；
- 答案能通过 add/del 一个元素维护；
- 查询之间可以改变顺序；
- 区间统计量不好用线段树合并；
- 需要维护众数、颜色种数、计数平方和、异或数对等信息。

## 更适合树状数组的情况

- 区间不同数个数可以按右端点离线；
- 单点修改 + 区间求和；
- 逆序对、排名统计；
- 操作可以转成前缀贡献。

例如：

```text
P1972 HH 的项链既可以用莫队，也可以用树状数组离线。
```

如果只求区间不同数个数，树状数组离线通常更快。

## 更适合线段树的情况

- 需要在线处理；
- 区间修改和查询有明确合并规则；
- lazy 标记容易维护；
- 查询顺序不能改变。

## 更适合前缀和的情况

- 区间和、区间异或等简单可减信息；
- 没有复杂统计；
- 不需要维护颜色种类或数对贡献。

## 1. 普通莫队

# 最终掌握标准

学完莫队后，看到一道题应该能快速判断：

1. 询问是否可以离线？
2. 是否是区间查询？
3. 当前答案能否通过加入/删除一个元素维护？
4. `add` 和 `del` 是否容易写，且是否互逆？
5. 是否需要离散化？
6. 是否涉及组合数、概率、异或、mex 或众数？
7. 是否存在修改操作？
8. 如果有修改，是否需要带修莫队？
9. 如果在树上，是否能用欧拉序转成区间？
10. 如果删除很难，是否可以用回滚莫队？

如果这些问题都能回答出来，说明你已经掌握了莫队的建模方式，而不是只会套普通莫队模板。
