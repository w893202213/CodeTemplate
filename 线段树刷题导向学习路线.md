# 一、不带懒标记的简单问题

## 学习目标

掌握以下基本能力：

- 建树 `build`
- 单点修改 `modify`
- 区间查询 `query`
- 信息上传 `pushup`
- 维护 `sum / max / min` 等基础信息

这一阶段不要急着学懒标记，先把线段树的递归结构写熟。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P1816 忠诚 | https://www.luogu.com.cn/problem/P1816 | 静态区间最小值查询 |
| 2 | 洛谷 P1531 I Hate It | https://www.luogu.com.cn/problem/P1531 | 单点更新 + 区间最大值查询 |
| 3 | 洛谷 P1198 [JSOI2008] 最大数 | https://www.luogu.com.cn/problem/P1198 | 动态插入 + 查询末尾区间最大值 |
| 4 | 力扣 307 区域和检索 - 数组可修改 | https://leetcode.cn/problems/range-sum-query-mutable/ | 单点修改 + 区间和查询，适合面试场景 |

## 本阶段建议模板

需要能默写：

```cpp
build();
pushup();
modify(pos, val);
query(l, r);
```

## 阶段总结

完成这一阶段后，你应该能独立解决：

- 区间最值查询
- 区间和查询
- 单点修改
- 动态追加元素后的区间查询

# 二、带懒标记的简单问题

## 学习目标

掌握以下能力：

- 区间修改
- 区间查询
- 懒标记 `lazy tag`
- 标记下传 `pushdown`
- 多种懒标记类型：加法、乘法、翻转、覆盖

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P3372 【模板】线段树 1 | https://www.luogu.com.cn/problem/P3372 | 区间加 + 区间求和，最标准 lazy 模板 |
| 2 | 洛谷 P3870 [TJOI2009] 开关 | https://www.luogu.com.cn/problem/P3870 | 区间翻转 + 查询 1 的个数，练异或型懒标记 |
| 3 | 洛谷 P2574 XOR 的艺术 | https://www.luogu.com.cn/problem/P2574 | 01 串区间翻转 + 区间 1 的数量查询 |
| 4 | 洛谷 P3373 【模板】线段树 2 | https://www.luogu.com.cn/problem/P3373 | 区间乘 + 区间加 + 区间求和 |
| 5 | 洛谷 P2023 [AHOI2009] 维护序列 | https://www.luogu.com.cn/problem/P2023 | 继续巩固乘法标记、加法标记的组合 |

## 本阶段重点

懒标记的核心逻辑是：

1. 对一个完整覆盖的区间，直接修改当前节点；
2. 同时记录懒标记；
3. 真正需要访问子节点时，再把标记下传；
4. 下传完成后清空当前节点标记。

## 常见错误

- 忘记 `pushdown`
- 懒标记叠加顺序写错
- 区间长度计算错误
- 乘法标记和加法标记组合错误
- 覆盖标记和加法标记优先级写反

# 三、中等难度问题

## 学习目标

这一阶段不再只维护简单的 `sum / max / min`，而是开始维护复杂区间信息。

重点是学会回答三个问题：

1. 节点应该维护什么？
2. 左右儿子的信息如何合并？
3. 是否需要懒标记？

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P4513 小白逛公园 | https://www.luogu.com.cn/problem/P4513 | 最大子段和，维护 `sum / lmax / rmax / ans` |
| 2 | 洛谷 P4145 上帝造题的七分钟 2 / 花神游历各国 | https://www.luogu.com.cn/problem/P4145 | 区间开方 + 区间求和，练剪枝与势能分析 |
| 3 | 洛谷 P1471 方差 | https://www.luogu.com.cn/problem/P1471 | 区间加 + 区间平均数/方差，维护 `sum` 和 `sum of squares` |
| 4 | 洛谷 P1253 扶苏的问题 | https://www.luogu.com.cn/problem/P1253 | 区间覆盖 + 区间加 + 区间最大值 |
| 5 | 力扣 699 掉落的方块 | https://leetcode.cn/problems/falling-squares/ | 离散化 + 区间查询最大值 + 区间覆盖 |

## 本阶段关键思想

### 1. 最大子段和类

节点一般维护：

```cpp
struct Node {
    long long sum;   // 区间总和
    long long lmax;  // 从左端点开始的最大子段和
    long long rmax;  // 从右端点结束的最大子段和
    long long ans;   // 区间内部最大子段和
};
```

合并时：

```cpp
res.sum = left.sum + right.sum;
res.lmax = max(left.lmax, left.sum + right.lmax);
res.rmax = max(right.rmax, right.sum + left.rmax);
res.ans = max({left.ans, right.ans, left.rmax + right.lmax});
```

### 2. 方差类

需要维护：

- 区间和 `sum`
- 区间平方和 `sum2`

因为：

```text
variance = average(x^2) - average(x)^2
```

### 3. 区间开方类

通常不能直接套普通懒标记，而是利用性质剪枝：

- 如果区间最大值和最小值变化一致，可以整体处理；
- 或者当区间元素都变成 0/1 后，不再递归修改；
- 本质是利用开平方后数值快速下降的性质。

# 每道题的复盘模板

建议每做完一道题，都写这三句话：

## 1. 节点维护什么信息？

例如：

```text
本题节点维护区间和 sum。
```

或者：

```text
本题节点维护 sum、lmax、rmax、ans 四个信息，用于合并最大子段和。
```

## 2. pushup 如何合并？

例如：

```text
父节点的 sum 等于左右儿子的 sum 之和。
```

或者：

```text
父节点的最大子段和可能来自左儿子、右儿子，或者跨越左右儿子的区间。
```

## 3. 是否需要 lazy？如果需要，标记如何处理？

例如：

```text
本题需要区间加，因此使用 add 懒标记。
完整覆盖时直接修改当前节点 sum，并累计 add。
递归访问子节点前下传 add。
```

# 学习建议

## 第一轮：只求 AC

第一轮刷题时，不要追求代码特别短。重点是：

- 能写出递归版线段树
- 能调通 `build / modify / query`
- 能理解 `pushup / pushdown`
- 能独立处理边界

## 第二轮：整理模板

完成基础题后，整理出至少 4 个模板：

1. 单点修改 + 区间查询模板
2. 区间加 + 区间和模板
3. 区间乘加 + 区间和模板
4. 区间翻转模板

## 第三轮：专题突破

中等题开始，不要只背模板，而要训练建模能力：

- 这道题维护什么？
- 是否可以合并？
- 是否支持懒标记？
- 修改操作是否满足区间整体处理？
- 如果不能 lazy，是否可以剪枝？

---

# 最终掌握标准

当你看到一道线段树题时，能快速判断：

1. 是普通线段树、懒标记线段树，还是变体？
2. 维护的是和、最值、计数、最大子段和，还是更复杂的信息？
3. 修改操作能否用懒标记？
4. 查询信息如何由左右儿子合并？
5. 是否需要离散化、动态开点、可持久化或扫描线？

如果这些问题都能回答出来，说明线段树已经从“会背模板”进入“会建模”的阶段。
