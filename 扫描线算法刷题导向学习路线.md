# 扫描线算法到底解决什么问题？

扫描线算法的核心思想是：

```text
把二维问题按照一个坐标轴排序，
让一条“扫描线”从左到右或从下到上移动，
在移动过程中维护另一维上的有效信息。
```

常见处理对象：

- 矩形面积并；
- 矩形周长并；
- 多矩形覆盖次数；
- 二维数点；
- 点被矩形覆盖次数；
- 固定窗口内最大权值；
- 区间事件排序；
- 线段交点统计；
- 几何覆盖类问题。

扫描线通常配合：

```text
排序 + 离散化 + 线段树 / 树状数组 / 差分
```

其中最经典的是：

```text
扫描线 + 线段树维护覆盖长度
```

# 一、基础入门：事件排序与一维扫描

## 学习目标

这一阶段先不急着写复杂线段树，而是理解“事件”的概念。

需要掌握：

- 把区间、矩形、点转化成事件；
- 按某个坐标排序；
- 遇到进入事件就加入贡献；
- 遇到离开事件就删除贡献；
- 用差分、树状数组或简单计数维护当前状态。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P1496 火烧赤壁 | https://www.luogu.com.cn/problem/P1496 | 一维区间覆盖长度，离散化 + 差分，是扫描线前置题 |
| 2 | 洛谷 P1884 [USACO12FEB] Overplanting S | https://www.luogu.com.cn/problem/P1884 | 矩形面积并的二维差分做法，可与扫描线对比 |
| 3 | 洛谷 P3397 地毯 | https://www.luogu.com.cn/problem/P3397 | 二维差分入门，帮助理解矩形覆盖 |
| 4 | 洛谷 P2163 [SHOI2007] 园丁的烦恼 | https://www.luogu.com.cn/problem/P2163 | 二维数点，扫描线 + 树状数组 |

## 事件的基本概念

例如一个一维区间：

```text
[l, r]
```

可以拆成两个事件：

```text
位置 l：加入一个覆盖
位置 r：删除一个覆盖
```

如果是半开区间 `[l, r)`，写起来更自然：

```text
l 处 +1
r 处 -1
```

扫描时维护当前覆盖次数：

```text
cover += event.delta
```

如果：

```text
cover > 0
```

说明当前这一段被覆盖。

## 二维数点中的扫描线

二维数点常见问题：

```text
给定若干点和若干矩形，询问每个矩形内有多少点。
```

可以转化成四个前缀查询：

```text
count(x2, y2)
- count(x1 - 1, y2)
- count(x2, y1 - 1)
+ count(x1 - 1, y1 - 1)
```

其中：

```text
count(x, y) = 横坐标 <= x 且纵坐标 <= y 的点数。
```

做法：

```text
1. 点按 x 排序；
2. 查询事件按 x 排序；
3. 扫描 x；
4. 把已经扫到的点的 y 坐标加入树状数组；
5. 查询 y 前缀点数。
```

## 本阶段常见错误

- 没有区分闭区间和半开区间；
- 事件排序时同坐标的加入/删除顺序没有想清楚；
- 二维数点中没有做容斥；
- y 坐标值域太大时忘记离散化；
- 使用扫描线时没有保存原询问编号。

# 二、标准模板：矩形面积并

## 学习目标

这一阶段掌握扫描线最经典模板：

```text
矩形面积并 = 扫描线 + 坐标离散化 + 线段树维护覆盖长度
```

需要掌握：

- 矩形拆成两条横边或竖边；
- 每条边变成一个事件；
- 事件按扫描方向排序；
- 线段树维护当前被覆盖的总长度；
- 面积增量 = 覆盖长度 × 扫描线移动距离；
- 离散化后线段树维护的是“坐标段”，不是单个点。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P5490【模板】扫描线 & 矩形面积并 | https://www.luogu.com.cn/problem/P5490 | 标准矩形面积并模板 |
| 2 | HDU 1542 Atlantis | https://acm.hdu.edu.cn/showproblem.php?pid=1542 | 矩形面积并经典英文题，坐标为小数 |
| 3 | 洛谷 P1884 [USACO12FEB] Overplanting S | https://www.luogu.com.cn/problem/P1884 | 可用二维差分，也可练扫描线写法 |
| 4 | 力扣 850 矩形面积 II | https://leetcode.cn/problems/rectangle-area-ii/ | 面试版矩形面积并，注意取模和长整型 |

## 矩形如何拆成事件？

一个矩形：

```text
(x1, y1), (x2, y2)
```

如果从下往上扫描，就拆成两条水平边：

```text
y = y1：在 [x1, x2) 加入覆盖，delta = +1
y = y2：在 [x1, x2) 删除覆盖，delta = -1
```

事件结构可以写成：

```cpp
struct Event {
    long long y;
    long long x1, x2;
    int delta;
};
```

按 `y` 从小到大排序。

## 面积如何累加？

设当前事件为 `event[i]`，上一个事件为 `event[i - 1]`。

在两条扫描线之间：

```text
高度 = event[i].y - event[i - 1].y
宽度 = 当前 x 轴上被覆盖的总长度
```

所以面积增量为：

```cpp
area += coveredLength * (event[i].y - event[i - 1].y);
```

然后再处理当前事件对覆盖状态的影响。

常见流程：

```cpp
sort(events.begin(), events.end());

for (int i = 0; i < events.size(); i++) {
    area += tree[1].len * (events[i].y - events[i - 1].y);
    update(events[i].x1, events[i].x2, events[i].delta);
}
```

实际代码中需要从 `i = 1` 开始，或者先处理第一个事件，避免访问越界。

## 线段树维护什么？

线段树节点通常维护：

```cpp
struct Node {
    int cover;          // 当前区间被完整覆盖了多少次
    long long len;      // 当前区间被覆盖的真实长度
};
```

pushup 规则：

```text
如果 cover > 0：
    len = xs[r + 1] - xs[l]
否则：
    如果是叶子：len = 0
    否则：len = 左儿子 len + 右儿子 len
```

注意：

```text
线段树维护的是 xs[i] 到 xs[i + 1] 之间的线段。
```

不是单个离散点。

## 本阶段常见错误

- 离散化后把点当成线段维护；
- 更新 `[x1, x2]` 而不是 `[x1, x2)` 对应的坐标段；
- `pushup` 中长度写成 `r - l + 1`，没有使用原坐标差；
- 面积累加顺序错误；
- 事件同坐标时处理顺序不当；
- 坐标最大到 `1e9`，面积可能超过 int；
- 力扣 850 中忘记对答案取模。

# 三、中等难度：周长并、覆盖次数与最大覆盖权值

## 学习目标

这一阶段不再只维护“覆盖长度”，而是维护更复杂的信息。

常见扩展：

- 矩形周长并；
- 覆盖次数达到 k 的长度；
- 被覆盖区域的连通段数；
- 最大覆盖权值；
- 固定窗口能覆盖的最大点权。

| 2 | HDU 1828 Picture | https://acm.hdu.edu.cn/showproblem.php?pid=1828 | 矩形周长并经典英文题 |
| 3 | 洛谷 P1502 窗口的星星 | https://www.luogu.com.cn/problem/P1502 | 点转矩形，扫描线 + 线段树维护最大值 |
| 4 | HDU 1255 覆盖的面积 | https://acm.hdu.edu.cn/showproblem.php?pid=1255 | 求被覆盖至少两次的面积 |

## 矩形周长并

面积并只需要维护：

```text
当前被覆盖的总长度 len
```

周长并还需要考虑边界变化。

常见维护信息：

```cpp
cover：覆盖次数
len：当前覆盖总长度
seg：当前覆盖形成了多少段
lcov：当前区间左端点是否被覆盖
rcov：当前区间右端点是否被覆盖
```

周长贡献通常分为两部分：

```text
横向边界贡献：覆盖长度变化量
纵向边界贡献：覆盖段数 × 扫描线移动距离 × 2
```

也就是说，每移动一段距离，每个覆盖段都会贡献左右两条竖边。

## 覆盖至少 k 次的面积

普通面积并统计：

```text
cover > 0 的长度
```

覆盖至少两次时，需要统计：

```text
cover >= 2 的长度
```

线段树节点可以维护：

```text
len1：覆盖至少 1 次的长度
len2：覆盖至少 2 次的长度
```

pushup 逻辑比普通面积并更复杂：

```text
如果 cover >= 2：
    len2 = 当前节点真实长度
如果 cover == 1：
    len2 = 左儿子 len1 + 右儿子 len1
如果 cover == 0：
    len2 = 左儿子 len2 + 右儿子 len2
```

类似地也可以扩展到覆盖至少 k 次。

## 窗口的星星

这类题通常可以把“一个点能被哪些窗口位置包含”转成“一个矩形贡献”。

如果一颗星星位于：

```text
(x, y)
```

窗口宽高为：

```text
W, H
```

那么窗口左下角如果落在某个范围内，就能看到这颗星星。

于是每个点可以转成一个带权矩形，然后问题变成：

```text
求所有带权矩形覆盖下的最大权值。
```

用扫描线处理一维坐标，线段树维护另一维的区间加和最大值。

## 本阶段常见错误

- 周长并中只算覆盖长度变化，漏掉段数贡献；
- 没有维护左右端点覆盖状态，导致段数计算错误；
- 覆盖至少两次时 pushup 分类不完整；
- 窗口星星题中边框是否包含没有处理好；
- 点转矩形时坐标范围写反；
- 带权最大覆盖题中线段树应维护最大值，而不是覆盖长度。

## 二维数点的标准套路

二维数点本质上是：

```text
把二维矩形查询拆成若干个前缀查询。
```

例如询问矩形：

```text
[x1, x2] × [y1, y2]
```

可以转化成：

```text
F(x2, y2)
- F(x1 - 1, y2)
- F(x2, y1 - 1)
+ F(x1 - 1, y1 - 1)
```

其中：

```text
F(x, y) = 横坐标 <= x 且纵坐标 <= y 的点数。
```

实现：

```text
1. 点按 x 排序；
2. 前缀查询事件也按 x 排序；
3. 扫描 x；
4. 把扫过的点的 y 加入树状数组；
5. 对每个查询事件统计 y 前缀。
```

## 扫描线 + set / 平衡树

有些题不是矩形覆盖，而是线段相交、最近线段、活动区间维护。

这时扫描线维护的不是覆盖长度，而是：

```text
当前与扫描线相交的对象集合。
```

常用结构：

```text
set
平衡树
优先队列
```

典型思路：

```text
1. 按 x 坐标处理所有事件；
2. 遇到线段左端点，将线段加入 set；
3. 遇到线段右端点，将线段删除；
4. 在 set 中检查相邻线段关系。
```

这类题比矩形面积并更偏计算几何，建议在掌握基础扫描线后再学。

## 扫描线与 CDQ 分治

扫描线和 CDQ 分治都常用于离线二维/三维偏序。

区别大致是：

```text
扫描线：按一个维度排序，动态维护另一个维度；
CDQ：通过分治处理时间/维度关系，再用树状数组统计另一维。
```

如果二维数点熟练后，可以继续学习：

```text
三维偏序 / CDQ 分治 / 陌上花开
```

代表题：

```text
洛谷 P3810【模板】三维偏序 / 陌上花开
https://www.luogu.com.cn/problem/P3810
```

## 本阶段常见错误

- 二维数点容斥符号写错；
- 查询事件没有保存原编号；
- 树状数组 y 坐标没有离散化；
- 点和查询事件同 x 时先后顺序错误；
- 线段交点类题中 set 比较函数依赖当前扫描线位置，写法不严谨；
- 把所有扫描线题都强行写成矩形面积并模板，忽视事件含义。

# 每道题的复盘模板

建议每做完一道扫描线题，都写下面几个问题。

## 1. 本题扫描哪个维度？

例如：

```text
本题按 y 坐标从下往上扫描。
```

或者：

```text
本题按 x 坐标从左往右扫描。
```

## 2. 事件是什么？

例如：

```text
每个矩形拆成两条水平边：
下边是加入事件，上边是删除事件。
```

或者：

```text
每个矩形查询被拆成四个前缀查询事件。
```

## 3. 扫描过程中维护什么？

例如：

```text
维护 x 轴上当前被覆盖的总长度。
```

或者：

```text
维护当前已经扫过的点的 y 坐标计数。
```

或者：

```text
维护当前 y 轴区间上的最大权值。
```

## 4. 是否需要离散化？

如果坐标范围很大，例如：

```text
0 <= x, y <= 10^9
```

通常需要离散化。

## 5. 线段树维护的是点还是区间段？

矩形面积并中，线段树维护的是：

```text
xs[i] 到 xs[i + 1] 之间的线段。
```

不是单个坐标点。

## 6. 答案如何累加？

例如面积并：

```text
答案 += 当前覆盖长度 × 扫描线移动距离。
```

周长并：

```text
答案 += 覆盖长度变化贡献 + 覆盖段数 × 移动距离 × 2。
```

二维数点：

```text
答案通过四个前缀查询容斥得到。
```

# 扫描线与其他算法的选择

## 更适合扫描线的情况

- 二维几何问题；
- 多个矩形的覆盖面积、周长、覆盖次数；
- 点和矩形的离线查询；
- 一维有序事件推进；
- 需要动态维护另一维信息；
- 坐标范围大但事件数量有限。

## 更适合二维差分的情况

- 坐标范围较小；
- 矩形覆盖后只需统计简单结果；
- 可以直接开二维数组；
- 不需要处理 `1e9` 级别坐标。

## 更适合树状数组离线的情况

- 二维数点；
- 点加入、矩形查询；
- 一维排序后另一维做前缀计数。

## 更适合线段树的情况

- 扫描线过程中需要维护覆盖长度、最大值、段数；
- 另一维需要区间修改和区间查询；
- 坐标需要离散化后维护区间段。

# 四类扫描线问题的区别

## 1. 一维事件扫描

代表题：

```text
P1496 火烧赤壁
```

核心：

```text
事件排序 + 覆盖计数。
```

## 2. 矩形面积并

代表题：

```text
P5490
HDU1542
LeetCode850
```

核心：

```text
扫描线 + 离散化 + 线段树维护覆盖长度。
```

## 3. 周长 / 多重覆盖 / 最大权值

代表题：

```text
P1856
HDU1255
P1502
```

核心：

```text
在线段树节点中维护更复杂的信息。
```

## 4. 二维数点与离线查询

代表题：

```text
P2163
P3755
P3810
```

核心：

```text
按一个维度排序，另一维用树状数组或线段树维护。
```

# 最终掌握标准

学完扫描线后，看到一道题应该能快速判断：

1. 能否把问题拆成事件？
2. 应该按 x 扫描还是按 y 扫描？
3. 每个对象会产生哪些加入/删除/查询事件？
4. 扫描过程中维护的是覆盖长度、覆盖次数、最大值，还是点数？
5. 是否需要离散化？
6. 线段树维护的是点还是相邻坐标段？
7. 答案是否通过“当前状态 × 扫描线移动距离”累加？
8. 是否需要容斥？
9. 是否有同坐标事件的先后顺序问题？
10. 是否可能用二维差分、树状数组离线或 CDQ 分治替代？

如果这些问题都能回答出来，说明你已经能把扫描线作为一种通用的离线建模方法，而不是只会背矩形面积并模板。
