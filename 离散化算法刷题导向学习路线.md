# 离散化解决什么问题？

离散化是一种数据预处理技巧，不是独立数据结构。

当原始值域很大，但真正出现的数据点很少时，可以把大范围数据映射成连续小下标。例如坐标范围是 `[-10^9, 10^9]`，但只出现了 `10^5` 个坐标点，就没必要开一个巨大数组。

常见用途：

1. 把大坐标映射成小下标；
2. 把大编号变量映射成并查集可处理的小编号；
3. 把大数值映射成排名，配合树状数组统计数量；
4. 在扫描线、矩形面积并中压缩坐标；
5. 在前缀和、差分、区间覆盖题中减少数组空间。

基础模板：

```cpp
vector<long long> xs;

sort(xs.begin(), xs.end());
xs.erase(unique(xs.begin(), xs.end()), xs.end());

int getId(long long x) {
    return lower_bound(xs.begin(), xs.end(), x) - xs.begin() + 1;
}
```

# 一、基础入门：排序去重与排名压缩

## 学习目标

掌握最基础的“排名离散化”。

例如：

```text
原数组：1000000000, -5, 100, -5
离散后：3, 1, 2, 1
```

这里只保留大小顺序和相等关系，不保留真实距离。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 力扣 1331 数组序号转换 | https://leetcode.cn/problems/rank-transform-of-an-array/ | 最纯粹的排名离散化 |
| 2 | 洛谷 P1059 明明的随机数 | https://www.luogu.com.cn/problem/P1059 | 排序、去重、输出有序结果 |
| 3 | 洛谷 P1097 统计数字 | https://www.luogu.com.cn/problem/P1097 | 大数值范围下统计出现次数 |
| 4 | AcWing 802 区间和 | https://www.acwing.com/problem/content/804/ | 大坐标点修改 + 区间查询，经典模板 |

## 本阶段必须掌握

离散化的四步：

```text
1. 收集所有会用到的值
2. 排序
3. 去重
4. 用 lower_bound 或 map 建立原值到新下标的映射
```

## 常见错误

- 忘记去重；
- 树状数组下标从 0 开始导致死循环；
- 只收集修改点，忘记收集查询端点；
- 把离散化编号差误当作真实距离；
- 重复元素没有映射到同一个编号。

# 二、区间离散化：差分、覆盖长度与面积

## 学习目标

这一阶段处理区间问题。重点是区分：

```text
排名离散化：只保留相对大小；
坐标离散化：需要保留相邻坐标之间的真实长度。
```

例如坐标：

```text
1, 10, 100
```

离散下标是：

```text
1, 2, 3
```

但真实线段长度是：

```text
10 - 1 = 9
100 - 10 = 90
```

如果题目求长度或面积，不能直接用离散化编号计算。

| 推荐顺序 | 题目 | 链接 | 训练点 |
|---|---|---|---|
| 1 | 洛谷 P1496 火烧赤壁 | https://www.luogu.com.cn/problem/P1496 | 一维区间覆盖长度，离散化 + 差分 |
| 2 | 洛谷 P1884 [USACO12FEB] Overplanting S | https://www.luogu.com.cn/problem/P1884 | 二维矩形覆盖面积，坐标离散化 + 二维差分 |
| 3 | 洛谷 P3397 地毯 | https://www.luogu.com.cn/problem/P3397 | 二维差分入门，可与 P1884 对比 |
| 4 | 洛谷 P3406 海底高铁 | https://www.luogu.com.cn/problem/P3406 | 路径区间统计，差分思想强化 |

## 一维区间覆盖长度

典型问题：

```text
给出若干区间 [l, r]，求被覆盖的总长度。
```

基本做法：

```text
1. 收集所有 l 和 r；
2. 排序去重；
3. 把区间映射到离散坐标；
4. 使用差分或标记数组统计覆盖情况；
5. 遍历相邻坐标段，如果该段被覆盖，则加上 xs[i + 1] - xs[i]。
```

## 二维矩形面积

典型问题：

```text
给出若干矩形，求被覆盖的总面积。
```

基本做法：

```text
1. 收集所有 x1、x2、y1、y2；
2. 分别对 x 坐标和 y 坐标排序去重；
3. 把每个矩形映射到离散坐标；
4. 使用二维差分或扫描线；
5. 统计被覆盖的小矩形面积。
```

面积计算必须使用真实坐标差：

```cpp
area += (xs[i + 1] - xs[i]) * (ys[j + 1] - ys[j]);
```

## 常见错误

- 用离散下标差代替真实长度；
- 没有区分“点”和“点之间的区间段”；
- 矩形面积统计时忘记使用原坐标差；
- 负坐标直接作为数组下标；
- 忽略闭区间、半开区间、端点定义的差异。

## 离散化 + 并查集

适用场景：

```text
变量编号很大，但实际出现的变量数量有限。
```

例如 P1955 中，变量编号可能达到 `10^9`，但一组数据里实际出现的变量只有 `O(n)` 个。

基本做法：

```text
1. 收集所有出现过的变量编号；
2. 排序去重；
3. 把原变量编号映射为 1 ~ cnt；
4. 用并查集处理相等关系；
5. 最后检查不等关系是否冲突。
```

注意：  
离散化只负责把大编号变小，真正判断矛盾的是并查集。

## 离散化 + 树状数组

适用场景：

```text
统计比当前数小、比当前数大、排名在某个范围内的元素数量。
```

逆序对经典写法：

```cpp
for (int i = n; i >= 1; i--) {
    int id = getId(a[i]);
    ans += query(id - 1);  // 右侧比 a[i] 小的数量
    add(id, 1);
}
```

## 离散化 + 前缀和

例如力扣 327：

```text
lower <= prefix[j] - prefix[i] <= upper
```

可以转化为：

```text
prefix[j] - upper <= prefix[i] <= prefix[j] - lower
```

此时需要把前缀和值、查询边界值一起离散化，再用树状数组统计数量。

## 常见错误

- 严格小于时查询 `id - 1`，小于等于时查询 `id`；
- 离散化时漏掉查询边界值；
- 树状数组下标为 0；
- P1955 这类题应先合并相等关系，再检查不等关系；
- `2 * nums[j]` 可能溢出，需要使用 `long long`。

## 扫描线中的离散化

矩形面积并常见做法：

```text
1. 将每个矩形拆成进入边和离开边；
2. 按扫描方向排序；
3. 对另一维坐标离散化；
4. 用线段树维护当前被覆盖的总长度；
5. 每移动一次扫描线，用“覆盖长度 × 移动距离”累加面积。
```

关键点：

```text
线段树维护的是相邻坐标之间的线段，不是单个坐标点。
```

例如：

```text
ys = [1, 4, 10]
```

实际线段是：

```text
[1, 4] 和 [4, 10]
```

而不是三个孤立点。

## CDQ 分治中的离散化

三维偏序常见形式：

```text
给出若干点 (a, b, c)，统计满足 a' <= a, b' <= b, c' <= c 的点数。
```

常见做法：

```text
1. 第一维排序；
2. 第二维用 CDQ 分治；
3. 第三维用树状数组统计；
4. 如果第三维值域大，需要先离散化。
```

## 奇偶约束中的离散化

对于区间 `[l, r]` 的奇偶信息，可以转成前缀和关系：

```text
sum[r] 与 sum[l - 1] 的奇偶关系
```

所以需要离散化的不是 `l` 和 `r` 本身，而是：

```text
l - 1 和 r
```

然后用带权并查集维护奇偶关系。

# 每道题的复盘模板

建议每做完一道离散化题，都写下面几个问题。

## 1. 为什么需要离散化？

例如：

```text
本题坐标范围达到 10^9，无法直接开数组，但实际出现的坐标数量只有 O(n)，因此需要离散化。
```

## 2. 离散化保留的是什么？

常见答案有三种：

```text
只保留大小顺序。
```

```text
只保留相等关系。
```

```text
保留坐标相邻段的真实长度。
```

## 3. 需要收集哪些值？

例如：

```text
本题需要收集所有修改位置和所有查询端点。
```

```text
本题需要收集所有矩形的 x1、x2、y1、y2。
```

```text
本题需要收集所有 l - 1 和 r，因为区间奇偶关系会转成前缀和关系。
```

## 4. 离散化后下标从几开始？

如果配合树状数组，建议从 1 开始：

```cpp
id = lower_bound(xs.begin(), xs.end(), x) - xs.begin() + 1;
```

如果只是数组或线段树，也可以从 0 开始，但前后要统一。

## 5. 是否需要用原坐标计算长度或面积？

如果题目涉及长度、面积、距离，一定不能直接用离散化编号差。

应使用：

```cpp
xs[i + 1] - xs[i]
```

或：

```cpp
(xs[i + 1] - xs[i]) * (ys[j + 1] - ys[j])
```

# 三类离散化的区别

## 1. 排名离散化

只保留大小关系。

适合：

- 排名；
- 逆序对；
- 第 k 小；
- 树状数组统计数量；
- 力扣 315；
- 洛谷 P1908。

## 2. 坐标离散化

保留坐标段之间的真实长度。

适合：

- 区间覆盖长度；
- 矩形面积并；
- 扫描线；
- 洛谷 P1496；
- 洛谷 P1884；
- 洛谷 P5490。

## 3. 哈希式离散化

只保留相等关系，不一定关心大小顺序。

适合：

- 并查集；
- 变量约束；
- 字符串编号；
- 洛谷 P1955。

# 最终掌握标准

学完离散化后，看到一道题应该能快速判断：

1. 数据值域是否过大？
2. 实际出现的数据量是否较少？
3. 需要保留大小顺序、相等关系，还是实际距离？
4. 是否要把查询端点、修改端点、`l - 1`、`r + 1` 等额外值加入离散化数组？
5. 离散化后是否要从 1 开始编号？
6. 是否需要配合树状数组、线段树、并查集、差分或扫描线？
7. 涉及长度和面积时，是否使用了原坐标差？

如果这些问题都能回答出来，说明你已经能够把离散化作为一种建模工具，而不是只会写 `sort + unique + lower_bound`。
